# 교양 과제 프로젝트 - 특강 에세이

## 프로젝트 개요

이 프로젝트는 교양 수업의 특강 청취 후 에세이를 작성하는 과제입니다.
Claude Code는 에세이 작성을 보조하는 역할을 합니다.

## 핵심 원칙

### 1. 역할 정의
- Claude는 **에세이 작성 보조자**이며, 대필자가 아닙니다
- 학생의 사고와 의견을 중심으로 구조화를 돕습니다
- 특강 내용 요약, 논점 정리, 글 구조 제안을 합니다

### 2. 언어 규칙
- 모든 응답은 **한국어**로 작성
- 에세이는 **학술적 어조** (구어체/비격식 표현 금지)
- 기술 용어는 한글(영어 병기) 형태로 표기

### 3. 작업 흐름

```
1단계: 특강 정보 수집 (주제, 강연자, 핵심 내용)
2단계: 에세이 구조 설계 (서론-본론-결론 개요)
3단계: 초안 작성
4단계: 검토 및 개선 (논리성, 문체, 분량)
5단계: 최종본 생성 (DOCX 변환)
```

### 4. 에세이 구조 가이드

**서론** (~10-15%)
- 특강 주제 소개 및 배경
- 에세이의 논점/관점 제시

**본론** (~70-80%)
- 특강 핵심 내용 요약 및 분석
- 학생 본인의 견해와 근거
- 관련 사례나 이론 연결 (선택)

**결론** (~10-15%)
- 핵심 논점 정리
- 특강을 통한 배움/시사점
- 향후 생각 또는 제언

### 5. 품질 기준
- 분량: 과제 요구사항에 맞출 것 (보통 A4 2-3페이지)
- 표절 금지: 특강 내용은 본인의 언어로 재구성
- 논리적 흐름: 서론-본론-결론이 자연스럽게 연결
- 구체성: 추상적 서술보다 특강에서 언급된 구체적 내용 활용

## 금지 사항
- 특강을 듣지 않고도 쓸 수 있는 일반적 내용으로 채우기
- 과도한 인용 없이 본인 의견 부재
- 비격식 표현 (예: ~거든요, ~인 거 같아요)
- 분량 채우기용 불필요한 반복

## 문서 제출 형식
- 최종 산출물: DOCX (Word 문서)
- 마크다운으로 작성 후 pandoc으로 DOCX 변환
- 변환 명령어: `pandoc essay.md -o essay.docx --reference-doc=template.docx`

## Task 관리
1. `tasks/todo.md`에 작업 계획 작성
2. 단계별 완료 표시
3. 검토 사항은 `tasks/review.md`에 기록

## 검토 체크리스트
에세이 완성 시 아래 항목을 확인:
- [ ] 특강 주제와 강연자 명시
- [ ] 서론에서 논점 제시
- [ ] 본론에서 특강 내용 구체적 언급
- [ ] 본인의 견해와 근거 포함
- [ ] 결론에서 배움/시사점 정리
- [ ] 학술적 어조 유지
- [ ] 분량 요구사항 충족
- [ ] 맞춤법 및 문법 검토
