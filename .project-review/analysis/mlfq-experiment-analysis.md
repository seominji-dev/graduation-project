## 원인 분석 (Root Cause Analysis)

### 현재 시뮬레이션의 한계

**문제:** MLFQ의 큐 이동 메커니즘이 작동하지 않음

**이유:** 현재 시뮬레이션은 **선점형(Non-preemptive)** 방식으로 작동
1. 모든 요청이 먼저 큐에 삽입됨
2. 요청이 하나씩 순차적으로 처리됨 (완전히 완료된 후 다음 요청)
3. 각 요청이 처리 시간을 모두 소진 후에 `feedback()`이 호출되지만, 이미 처리가 완료되었으므로 큐 이동이 의미 없음

### MLFQ 큐 할당량

```
Q0: 1,000ms (1초)  → Short 요청(171ms), Medium 요청의 일부
Q1: 3,000ms (3초)  → Medium 요청(1,045ms)
Q2: 8,000ms (8초)  → Long 요청(3,501ms)
Q3: 무제한          → 매우 긴 요청
```

**이론적으로:** Medium 요청(1,045ms)과 Long 요청(3,501ms)은 Q0의 할당량(1,000ms)을 초과하여 Q1/Q2로 이동해야 함

**실제:** 선점형 시뮬레이션에서는 요청이 중간에 중단되지 않으므로 큐 이동 발생 안 함

---

## 해결 방안 (Solution)

### 시간 분할(Time-sliced) 시뮬레이션 필요

진정한 MLFQ의 적응형 특성을 입증하려면:

1. **타임 퀀텀(500ms)** 단위로 처리
2. **타임 퀀텟 경과 시:** 요청이 완료되지 않았으면 큐 강등 (Q0 → Q1 → Q2)
3. **다음 요청 처리:** 상위 큐의 요청 우선 처리

### 예상 결과

시간 분할 시뮬레이션 시:

| 카테고리 | FCFS 평균 대기 | MLFQ 평균 대기 | 개선율 |
|---------|--------------|--------------|--------|
| Short | ~2,000ms | ~500ms | **~75%** |
| Medium | ~5,000ms | ~2,000ms | **~60%** |
| Long | ~8,000ms | ~5,000ms | **~37%** |

## 결론

### 현재 상태
- **한계점 인식됨:** 평가 보고서에서 지적된 대로 MLFQ 실험이 FCFS와 동일한 결과
- **원인 파악됨:** 선점형 시뮬레이션으로는 MLFQ의 적응형 특성 발현 불가

### 추후 작업
1. **시간 분할 시뮬레이션 구현:** 타임 퀀텀 단위 처리 및 선점형 큐 이동
2. **실험 결과 업데이트:** 시간 분할 방식의 MLFQ vs FCFS 비교
3. **논문/발표 자료 수정:** MLFQ 적응형 성능 입증 결과 추가

### 잠정적 해결책
**학부 졸업 프로젝트 수준에서 시간 분할 시뮬레이션의 복잡도를 고려하여:**

1. **한계점 솔직하게 인지:** 현재 시뮬레이션의 선점형 특성으로 MLFQ 적응형 특성이 발현하지 않음을 명시
2. **이론적 근거 설명:** MLFQ Rule 4(시간 할당량 초과 시 큐 강등)의 정확한 구현을 코드로 증명
3. **향후 연구 제시:** 시간 분할 시뮬레이션을 통한 MLFQ 적응형 성능 입증은 향후 연구 과제로 제안
