## 1. 기능 요구사항

### FR-1: 스케줄링 알고리즘 구현

#### FR-1.1: FCFS (First-Come, First-Served) 스케줄러
- **우선순위**: 필수 (P0)
- **설명**: 도착 순서대로 요청을 처리하는 스케줄러 (베이스라인)
- **상세 요구사항**:
  - FR-1.1.1: 요청 도착 순서를 타임스탬프 기반으로 보장
  - FR-1.1.2: FIFO 큐 구조 사용
  - FR-1.1.3: 비선점형 스케줄링 구현
- **검증 기준**: 10개 요청 제출 시 타임스탬프 순서대로 처리 완료
- **[개선됨] 실제 달성**: 100% 테스트 커버리지, 100% 통과

#### FR-1.2: Priority Scheduling 스케줄러
- **우선순위**: 필수 (P0)
- **설명**: 우선순위 기반으로 요청을 처리하는 스케줄러
- **상세 요구사항**:
  - FR-1.2.1: 4단계 우선순위 지원 (URGENT > HIGH > NORMAL > LOW)
  - FR-1.2.2: [개선됨] **비선점형** 스케줄링 구현
  - FR-1.2.3: Aging 메커니즘으로 기아 방지
- **검증 기준**: URGENT 요청이 LOW 요청보다 먼저 처리됨 (도착 순서 무관)
- **[개선됨] 실제 달성**: URGENT 요청이 FCFS 대비 62% 빠름 (Cohen's d=0.78, p<0.001)

#### FR-1.3: MLFQ (Multi-Level Feedback Queue) 스케줄러
- **우선순위**: 필수 (P0)
- **설명**: 다단계 피드백 큐 기반 스케줄러
- **상세 요구사항**:
  - FR-1.3.1: 4단계 큐 구현 (Q0, Q1, Q2, Q3)
  - FR-1.3.2: 큐별 타임 퀀텀 설정 (Q0: 500ms, Q1: 1500ms, Q2: 4000ms, Q3: 무제한)
  - FR-1.3.3: MLFQ 5가지 규칙 구현
  - FR-1.3.4: 주기적 Boost 메커니즘
  - **[개선됨] FR-1.3.5**: 시간 슬라이스(500ms) 기반 **선점형** 모드 지원
  - **[개선됨] FR-1.3.6**: 동시 요청 경쟁 환경에서의 Short 요청 최적화
- **검증 기준**: 짧은 작업은 Q0에서 완료, 긴 작업은 하위 큐로 강등
- **[개선됨] 실제 달성**: Short 요청 81.14% 개선 (동시 경쟁 실험)

#### FR-1.4: WFQ (Weighted Fair Queuing) 스케줄러
- **우선순위**: 필수 (P0)
- **설명**: 가중치 기반 공정 스케줄링
- **상세 요구사항**:
  - FR-1.4.1: 테넌트별 가중치 관리
  - FR-1.4.2: Virtual Finish Time 계산
  - FR-1.4.3: 테넌트 티어별 기본 가중치 (enterprise: 100, premium: 50, standard: 10, free: 1)
  - **[개선됨] FR-1.4.4**: 이중 수준 JFI 측정 (시스템 수준 + 테넌트 수준)
- **검증 기준**: 가중치 비율에 비례하는 대기시간 차등화
- **[개선됨] 실제 달성**: Enterprise가 Free 대비 5.8배 빠름, 시스템 JFI 0.89

#### FR-1.5: [개선됨] Rate Limiter 스케줄러
- **우선순위**: 필수 (P0)
- **설명**: 토큰 버킷 알고리즘 기반 속도 제한
- **상세 요구사항**:
  - FR-1.5.1: 테넌트별 토큰 속도 설정
  - FR-1.5.2: 버스트 용량 제어
  - FR-1.5.3: 토큰 고갈 시 요청 대기
- **검증 기준**: 설정된 속도를 초과하는 요청이 제한됨
- **실제 달성**: 10,000 요청 실험에서 안정적인 속도 제어

### FR-2: REST API

#### FR-2.1: 요청 관리 API
- **우선순위**: 필수 (P0)
- **엔드포인트**:
  - `POST /requests` - 새 요청 제출
  - `GET /requests/:id` - 요청 상태 조회
  - `GET /requests` - 전체 요청 목록
- **입력 검증**:
  - prompt: 필수, 문자열
  - priority: 선택, enum (LOW, NORMAL, HIGH, URGENT)
  - tenantId: 선택 (WFQ 사용 시 필수)

#### FR-2.2: 스케줄러 관리 API
- **우선순위**: 필수 (P0)
- **엔드포인트**:
  - `POST /scheduler/switch` - 알고리즘 변경
  - `GET /scheduler/status` - 스케줄러 상태
  - `GET /scheduler/stats` - 통계 조회
- **지원 알고리즘**: fcfs, priority, mlfq, wfq, rateLimiter

#### FR-2.3: [개선됨] 통계 및 공정성 API
- **우선순위**: 필수 (P0)
- **엔드포인트**:
  - `GET /api/stats` - 전체 통계
  - `GET /api/stats/tenant/:id` - 테넌트별 통계
  - `GET /api/fairness` - 공정성 지표 (JFI)
  - `GET /api/logs` - 요청 로그
  - `GET /api/health` - 헬스 체크

## 부록: 개선 내용 요약

| 영역 | 원본 요구사항 | 개선안 | 개선 이유 |
|------|---------------|--------|-----------|

| API | 기본 API | 통계/공정성 API 추가 | 분석 기능 강화 |
