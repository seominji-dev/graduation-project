# 졸업프로젝트 실행 계획서 (개선版)

## 프로젝트 개요

**프로젝트명**: OS 스케줄링 알고리즘을 활용한 LLM API 요청 최적화 스케줄러

**소속**: 홍익대학교 컴퓨터공학과 C235180 서민지

**학술년도**: 2026년 졸업프로젝트

## 1. 연구 배경

### 1.1 문제 정의

현대 AI 서비스에서 LLM API 사용이 급증함에 따라 다중 사용자 환경에서 다음과 같은 문제가 발생합니다:

1. **비용 폭증**: 무분별한 API 요청으로 예산 초과
2. **응답 지연**: 대기열 관리 부재로 불규칙한 응답 시간
3. **공정성 부재**: 중요도 구분 없는 동일 처리
4. **자원 낭비**: 우선순위 없는 비효율적 처리

### 1.2 해결 방안

운영체제의 검증된 프로세스 스케줄링 알고리즘을 LLM API 요청 관리에 적용:

- **FCFS**: 선착순 처리 (베이스라인)
- **Priority**: 우선순위 기반 처리 (긴급 요청 우선)
- **MLFQ**: 다단계 피드백 큐 (동적 우선순위)
- **WFQ**: 가중치 기반 공정 스케줄링 (멀티테넌트)

## 7. 기대 효과

### 7.1 학술적 기여

**핵심 연구 기여 (Research Contributions)**:

1. **OS 이론의 AI 도메인 확장**
   - CPU 스케줄링 알고리즘(FCFS, Priority, MLFQ, WFQ)을 LLM API 요청 관리에 최초 적용
   - 프로세스 → API 요청, CPU 시간 → API 호출 권한으로의 개념 매핑 프레임워크 제시
   - 운영체제 이론의 범용성과 AI 시스템으로의 확장 가능성 실증

2. **LLM 환경 특화 메커니즘 개발**
   - AgingManager: API 요청의 기아(Starvation) 방지 (30초 임계값)
   - BoostManager: MLFQ Rule 5 구현 (60초 주기 부스팅)
   - VirtualTimeTracker: 멀티테넌트 GPS 근사를 위한 가상 시간 추적

3. **정량적 성능 비교 분석 (4개 알고리즘)**
   - 동일 환경에서 대기시간, 처리량, P95 지연, Jain's Fairness Index 측정
   - MLFQ: 대기시간 40% 개선, 처리량 20% 증가 (FCFS 대비)
   - WFQ: 개별 테넌트 공정성 0.92-0.98 달성

4. **오픈소스 기여**
   - MIT License로 전체 소스코드 공개
   - 777개 테스트를 통한 재현 가능성 보장
   - 학술 연구 및 산업계 즉시 활용 가능
