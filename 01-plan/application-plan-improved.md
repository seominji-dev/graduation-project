# 졸업프로젝트 실행 계획서 (개선版)

## 프로젝트 개요

**프로젝트명**: OS 스케줄링 알고리즘을 활용한 LLM API 요청 최적화 스케줄러

**소속**: 홍익대학교 컴퓨터공학과 C235180 서민지

**학술년도**: 2025년 졸업프로젝트

## 1. 연구 배경

### 1.1 문제 정의

현대 AI 서비스에서 LLM API 사용이 급증함에 따라 다중 사용자 환경에서 다음과 같은 문제가 발생합니다:

1. **비용 폭증**: 무분별한 API 요청으로 예산 초과
2. **응답 지연**: 대기열 관리 부재로 불규칙한 응답 시간
3. **공정성 부재**: 중요도 구분 없는 동일 처리
4. **자원 낭비**: 우선순위 없는 비효율적 처리

### 1.2 해결 방안

운영체제의 검증된 프로세스 스케줄링 알고리즘을 LLM API 요청 관리에 적용:

- **FCFS**: 선착순 처리 (베이스라인)
- **Priority**: 우선순위 기반 처리 (긴급 요청 우선)
- **MLFQ**: 다단계 피드백 큐 (동적 우선순위)
- **WFQ**: 가중치 기반 공정 스케줄링 (멀티테넌트)

## 5. 품질 관리 계획

### 5.1 TRUST 5 Framework

- **Tested**: 85%+ 커버리지, 단위/통합 테스트
- **Readable**: 명확한 네이밍, 영어 주석
- **Unified**: ESLint/Prettier 스타일
- **Secured**: Zod 검증, 에러 처리
- **Trackable**: 구조화된 로깅

### 5.2 CI/CD 파이프라인

1. 코드 커밋 → Lint 검사
2. Lint 통과 → 테스트 실행
3. 테스트 통과 → 커버리지 확인
4. 커버리지达标 → 빌드
5. 빌드 성공 → 배포
