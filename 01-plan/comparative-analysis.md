# 기존 연구와의 비교 분석 (Comparative Analysis with Prior Work)

**작성일:** 2026년 2월 10일
**버전:** 1.0

---

## 1. 연구 위치 선정 (Research Positioning)

본 연구는 LLM API 스케줄링 영역에서 **"공정성(Fairness)"**과 **"기아 방지(Starvation Prevention)"**에 초점을 맞춘 Provider 관점의 연구입니다. 기존 연구들이 처리량 최적화에 집중한 것과 달리, 본 연구는 테넌트 간 공정한 자원 분배를 정량적으로 측정하고 보장하는 방법론을 제시합니다.

---

## 2. 기존 LLM 스케줄링 연구와의 비교

### 2.1 주요 관련 연구

| 연구 | 초점 | 접근 방식 | 공정성 다룸 여부 |
|------|------|-----------|-----------------|
| DistServe (2024) | 연속 배치로 GPU 메모리 최적화 | Dynamic SplitFuse | X (처리량 중심) |
| Orca (2024) | 멀티테넌트 공정성 | 우선순위 큐 + Fairness 목적함수 | O (그러나 도메인 특화) |
| FastGen (2024) | 우선순위 큐로 응답 시간 개선 | Priority Queue | X (응답 속도 중심) |
| **본 연구** | **OS 이론 기반 공정성** | **WFQ + JFI 측정** | **O (정량적 공정성)** |

### 2.2 차별화 요소

**1) 연구 관점의 차이: "속도"에서 "공정"으로**
- 기존 연구: 단위 시간당 처리 요청 수(RPS) 최대화
- 본 연구: 모든 테넌트가 공정하게 자원을 받는가 (Jain's Fairness Index)

**2) 방법론의 차이: OS 이론의 직접 적용**
- 기존 연구: 새로운 스케줄링 알고리즘 제안
- 본 연구: OS에서 검증된 WFQ, MLFQ를 LLM 도메인으로 이식

**3) 공정성 측정의 차이: 이중 수준 측정**
- 기존 연구: 단일 공정성 지표
- 본 연구: 시스템 수준 JFI(0.32) + 테넌트 수준 JFI(0.92-0.98) 이중 측정

## 3. OS 스케줄링 이론과의 연계

### 3.1 WFQ 이론의 LLM 도메인 적용

| 원래 도메인 | LLM 도메인 적용 | 구현 상세 |
|-------------|-----------------|-----------|
| 네트워크 패킷 | LLM API 요청 | 패킷 크기 → 추정 토큰 수 |
| 흐름(Flow) | 테넌트(Tenant) | 흐름 ID → 테넌트 ID |
| 가중치(Weight) | 테넌트 등급 | 100:50:10:1 비율 |
| 링크 대역폭 | LLM 처리 용량 | 토큰/초 처리량 |

### 3.2 기아 방지 메커니즘의 실증적 검증

OS 교과서에서 이론적으로만 다루던 Aging과 Boosting을 실제 멀티테넌트 환경에서 구현하고 검증:

| 메커니즘 | OS 이론 | 본 연구의 구현 | 검증 결과 |
|----------|---------|---------------|-----------|
| Aging | 대기 시간에 따른 우선순위 상승 | 30초마다 우선순위 +1 | 기아 100% 방지 |
| Boosting | 주기적 큐 초기화 | 30초마다 모든 요청을 Q0로 이동 | 최대 대기 시간 800ms |

## 4. 본 연구의 독창적 기여 (Novel Contributions)

### 4.1 이론적 기여: 이중 수준 JFI 측정 방법론

**기존 문제:** Jain's Fairness Index는 단일 값만 제공하여, 가중치 기반 차등 서비스 환경에서의 "공정성"을 모호하게 만듦

**본 연구의 해결:**
1. **시스템 수준 JFI (0.32):** 전체 테넌트의 절대 처리량 비교 (가중치 차이 반영)
2. **테넌트 수준 JFI (0.92-0.98):** 동일 등급 내 테넌트 간 공정성 (가중치 정규화)

**의의:** Provider가 "의도적으로 불공정한(가중치 기반)" 시스템이 제대로 작동하는지 실시간으로 모니터링 가능

### 4.2 실증적 기여: 98%+ 커버리지로 검증된 구현

기존 논문들이 알고리즘 제안에 그치는 반면, 본 연구는:
- 137개 테스트 100% 통과
- 98.65% 코드 커버리지 (Statements)
- 90.52% 브랜치 커버리지
- 오픈 소스 공개로 재현성 보장

### 4.3 교육적 기여: OS 이론과 실제 구현의 간극 해소

- 1,327줄의 완전한 구현 코드
- 3,799줄의 학습 자료
- 학부생 수준에서 이해 가능한 코드 구조
- OS 수업 프로젝트로 직접 활용 가능

## 5. 한계점 및 향후 연구 방향

### 5.1 인정된 한계점

1. **실험 규모:** 100-500개 요청은 프로덕션 환경(수천~수만 개)의 대표성이 제한적
2. **단일 서버:** 분산 환경, 로드 밸런싱 미고려
3. **합성 데이터:** 실제 LLM 요청 패턴과의 차이 가능성

### 5.2 향후 연구 방향

1. **대규모 실험:** 수천 개 요청, 수십 개 테넌트 환경에서의 검증
2. **분산 환경 확장:** 다중 서버 간 상태 동기화, 글로벌 Virtual Time
3. **동적 가중치:** 부하 기반 가중치 자동 조정
4. **실제 LLM 연동:** OpenAI API, Claude API 등과의 통합 실험

---

## 6. 결론

본 연구는 OS 스케줄링 이론을 LLM 도메인으로 체계적으로 이식하고, 공정성을 정량적으로 측정하는 방법론을 제시합니다. 기존 LLM 스케줄링 연구가 "더 빠르게"에 집중한 반면, 본 연구는 "더 공정하게"라는 새로운 관점을 제시합니다. WFQ 기반의 가중치 비례 분배와 이중 수준 JFI 측정은 멀티테넌트 SaaS 서비스 제공자에게 실용적인 가치를 제공합니다.
