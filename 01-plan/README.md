# 01-plan: 프로젝트 계획 문서

## 개요

이 폴더는 졸업프로젝트의 계획 단계 문서들을 포함합니다.

## 기준 문서

- **[수강신청서-내용-최종본.md](./수강신청서-내용-최종본.md)** - 대학 제출용 수강신청서의 최종 기준 문서 (Single Source of Truth)

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트 제목 | OS 스케줄링 알고리즘을 활용한 다중 사용자 LLM API 요청 관리 시스템 |
| 학번 | C235180 |
| 성명 | 서민지 |
| 학과 | 컴퓨터공학과 |

## 핵심 알고리즘

| 알고리즘 | 설명 | OS 수업 연관 개념 |
|----------|------|------------------|
| FCFS | 선착순 처리 (베이스라인) | 기본 스케줄링 |
| Priority | 긴급 요청 우선 처리 + Aging으로 기아 방지 | 우선순위 스케줄링, 노화 기법 |
| MLFQ | 요청 길이에 따른 동적 큐 이동 | 다단계 피드백 큐 |
| WFQ | 테넌트 가중치 기반 공정 배분 | Fair-Share Scheduling |

## 기술 스택

| 분류 | 기술 |
|------|------|
| 언어/런타임 | JavaScript (ES2024) / Node.js 22+ LTS |
| 웹 프레임워크 | Express.js 4.x |
| 데이터 저장 | JSON 파일 |
| 테스트 | Jest 29.x 이상 |
| LLM | Ollama (로컬) |

## 품질 목표

- 테스트 커버리지: 85% 이상
- 테스트 통과율: 100%
- 전체 코드: 약 1,000줄

## 연구 질문

| 연구 질문 | 알고리즘 | 핵심 탐구 내용 |
|----------|----------|---------------|
| RQ1 | Priority | URGENT 요청은 낮은 우선순위 요청보다 얼마나 빠르게 처리되는가? |
| RQ2 | MLFQ | 다양한 길이의 작업이 혼재된 환경에서 어떤 적응성을 보이는가? |
| RQ3 | WFQ | 가중치에 비례하는 서비스 차등화를 달성하는가? |
