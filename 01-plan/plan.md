# 졸업프로젝트 실행 계획서

## 프로젝트 개요

**프로젝트명**: OS 스케줄링 알고리즘을 활용한 LLM API 요청 최적화 스케줄러

**소속**: 홍익대학교 컴퓨터공학과 C235180 서민지

**학술년도**: 2026년 졸업프로젝트

---

## 1. 연구 배경

### 1.1 문제 정의

현대 AI 서비스에서 LLM(Large Language Model) API 사용이 급증함에 따라 다중 사용자 환경에서 다음과 같은 문제가 발생합니다:

1. **비용 폭증**: 무분별한 API 요청으로 예산 초과
2. **응답 지연**: 대기열 관리 부재로 불규칙한 응답 시간
3. **공정성 부재**: 중요도 구분 없는 동일 처리
4. **자원 낭비**: 우선순위 없는 비효율적 처리

### 1.2 해결 방안

운영체제 수업에서 배운 프로세스 스케줄링 알고리즘을 LLM API 요청 관리에 적용합니다:

| 알고리즘 | 설명 | OS 수업 연관성 |
|----------|------|---------------|
| **FCFS** | 선착순 처리 (베이스라인) | 가장 단순한 스케줄링 |
| **Priority** | 우선순위 기반 처리 | 긴급 요청 우선, Aging 기법 |
| **MLFQ** | 다단계 피드백 큐 | 동적 우선순위 조정 |
| **WFQ** | 가중치 기반 공정 스케줄링 | 멀티테넌트 공정성 |

## 6. 스케줄링 알고리즘 설계

### 6.1 FCFS (First-Come, First-Served)

**개념**: 먼저 도착한 요청을 먼저 처리

```javascript
class FCFSScheduler {
  constructor() {
    this.queue = [];
  }

  schedule(request) {
    this.queue.push(request);  // 큐 끝에 추가
    return request.id;
  }

  getNext() {
    return this.queue.shift(); // 큐 앞에서 꺼냄
  }
}
```

**장점**: 구현이 간단, 공정함
**단점**: 긴 요청이 뒤의 짧은 요청을 지연시킴 (Convoy Effect)

### 6.2 Priority Scheduling

**개념**: 우선순위가 높은 요청을 먼저 처리

| 우선순위 | 레벨 | 설명 |
|----------|------|------|
| URGENT | 3 | 즉시 처리 필요 |
| HIGH | 2 | 중요한 요청 |
| NORMAL | 1 | 일반 요청 |
| LOW | 0 | 낮은 우선순위 |

**Aging 메커니즘**: 오래 대기한 요청의 우선순위를 점진적으로 높여 기아(Starvation) 방지

```javascript
// 2분 이상 대기 시 우선순위 1단계 상승
if (waitTime > 120000 && request.priority < 3) {
  request.priority += 1;
}
```

### 6.3 MLFQ (Multi-Level Feedback Queue)

**개념**: 여러 우선순위 큐를 사용하고, 요청의 행동에 따라 큐 이동

| 큐 | Time Quantum | 대상 |
|----|--------------|------|
| Q0 | 1초 | 새 요청, 짧은 요청 |
| Q1 | 3초 | Q0 초과 요청 |
| Q2 | 8초 | Q1 초과 요청 |
| Q3 | 무제한 | 긴 요청 |

**MLFQ 5가지 규칙**:
1. Q0 > Q1 > Q2 > Q3 우선순위
2. Time Quantum 초과 시 하위 큐로 강등
3. 새 요청은 Q0에서 시작
4. 주기적 Boost (모든 요청을 Q0로 이동)
5. Aging으로 기아 방지

### 6.4 WFQ (Weighted Fair Queuing)

**개념**: 테넌트별 가중치에 따라 공정하게 자원 배분

| 티어 | 가중치 | 대역폭 비율 |
|------|--------|------------|
| Enterprise | 100 | 100/121 = 83% |
| Premium | 10 | 10/121 = 8% |
| Standard | 10 | 10/121 = 8% |
| Free | 1 | 1/121 = 1% |

**가상 완료 시간 계산**:
```
VirtualFinishTime = max(현재시간, 이전완료시간) + (처리시간 / 가중치)
```

### 7.1 학술적 기여

1. **OS 이론의 실제 적용**
   - CPU 스케줄링 알고리즘을 LLM API 관리에 적용
   - 프로세스 -> API 요청 개념 매핑 실증

2. **4가지 알고리즘 성능 비교**
   - 동일 환경에서 대기시간, 처리량 측정
   - 알고리즘별 장단점 분석

3. **학부생 수준 구현 가능성 검증**
   - 복잡한 인프라 없이 핵심 알고리즘 구현
   - 교육용 레퍼런스 제공
