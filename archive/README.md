# 아카이브 (Archive)

프로젝트 선정 과정에서 검토된 후보 프로젝트 및 참고 자료를 보관하는 폴더입니다.

**주의:** 이 폴더의 내용은 최종 프로젝트(LLM Scheduler)에 포함되지 않습니다.

---

## 폴더 구조

```
archive/
├── candidate-2-memory-manager/    # 후보 2: 메모리 관리자

```

| 2 | Memory Manager | 가상 메모리/페이징 | 아카이브 |

---

## 선정 이유

LLM Scheduler가 최종 선정된 이유:

1. **실용성:** LLM API 비용 절감이라는 실제 문제 해결
2. **적용 범위:** 4개의 스케줄링 알고리즘 구현 가능
3. **차별성:** 기존 연구와 차별화된 접근법
4. **완성도:** 테스트, 문서화, 실험까지 완료 가능
