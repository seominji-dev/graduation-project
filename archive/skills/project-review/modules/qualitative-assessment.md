# Qualitative Assessment Module

정성적 평가 모듈 - AI 기반의 깊이 있는 분석을 수행합니다.

## 정성적 평가의 철학

정성적 평가는 수치화하기 어려운 프로젝트의 본질적 가치를 평가합니다. 단순한 "잘함/못함" 판단을 넘어, **왜** 그런 평가가 나오는지에 대한 근거와 구체적인 개선 방향을 제시합니다.

### 평가 원칙

1. **근거 기반 (Evidence-Based):** 모든 평가는 구체적인 근거(문서 내용, 코드, 발표 자료)를 기반으로 합니다.
2. **맥락 고려 (Context-Aware):** 프로젝트의 목적, 범위, 제약 조건을 고려합니다.
3. **건설적 피드백 (Constructive):** 단점 지적뿐만 아니라 구체적 개선 방안을 제시합니다.
4. **균형 잡힌 시각 (Balanced):** 강점과 약점을 모두 식별합니다.

## AI 기반 분석 방법론

### 1. 문서 분석 프롬프트 전략

**연구 질의(RQ) 추출 및 평가:**
```
다음 논문/보고서에서 연구 질의(Research Question)를 추출하고 평가하세요:

1. RQ 식별: 명시적 또는 암시적 RQ를 모두 추출
2. 명확성 평가: RQ가 구체적이고 명확한가? (1-5점)
3. 실질성 평가: RQ가 실질적인 문제를 다루는가? (1-5점)
4. 실현 가능성 평가: RQ가 실현 가능한 범위인가? (1-5점)

출력 형식:
- RQ: [추출된 질의]
- 명확성: X/5 - [근거]
- 실질성: X/5 - [근거]
- 실현 가능성: X/5 - [근거]
```

**문헌 조사 충실성 분석:**
```
다음 문헌 조사 섹션을 분석하여 평가하세요:

1. 최신성: 최근 3-5년 연구 포함 여부
2. 포괄성: 해당 분야 주요 연구 망라 여부
3. 비교 분석: 단순 나열이 아닌 비교/분석 포함 여부
4. 간극 식별: 연구 간극(gap) 식별 여부

출력 형식:
- 최신성: X/5 - [근거]
- 포괄성: X/5 - [근거]
- 비교 분석: X/5 - [근거]
- 간극 식별: X/5 - [근거]
```

**방법론 타당성 분석:**
```
다음 방법론 섹션을 분석하여 평가하세요:

1. 근거 명시: 선택한 방법의 근거가 명확한가?
2. 대안 고려: 대안 방법과의 비교가 있는가?
3. 한계 인식: 방법의 한계를 인지하는가?
4. 적절성: 연구 목적에 부합하는가?

출력 형식:
- 근거 명시: X/5 - [근거]
- 대안 고려: X/5 - [근거]
- 한계 인식: X/5 - [근거]
- 적절성: X/5 - [근거]
```

### 2. 발표 자료 분석 프롬프트

**시각적 계층 구조 분석:**
```
다음 발표 슬라이드를 분석하여 평가하세요:

1. 핵심 메시지: 슬라이드당 핵심 메시지가 명확한가?
2. 정보 우선순위: 시각적 우선순위가 적절한가?
3. 여백과 레이아웃: 균형 잡혀 있는가?
4. 시각적 일관성: 디자인이 일관된가?

출력 형식:
- 핵심 메시지: X/5 - [근거]
- 정보 우선순위: X/5 - [근거]
- 여백과 레이아웃: X/5 - [근거]
- 시각적 일관성: X/5 - [근거]
```

**스토리텔링 흐름 분석:**
```
다음 발표 자료의 스토리텔링을 분석하여 평가하세요:

1. 구조: 도입-전개-결론의 구조가 명확한가?
2. 호기심 유발: 청중의 호기심을 유발하는가?
3. 기술 명확성: 기술적 내용이 명확하게 전달되는가?
4. 시간 관리: 내용이 주어진 시간에 적절한가?

출력 형식:
- 구조: X/5 - [근거]
- 호기심 유발: X/5 - [근거]
- 기술 명확성: X/5 - [근거]
- 시간 관리: X/5 - [근거]
```

### 3. 코드 품질 분석 프롬프트

**아키텍처 평가:**
```
다음 프로젝트의 아키텍처를 분석하여 평가하세요:

1. 모듈화: 모듈 간 결합도가 낮고 응집도가 높은가?
2. 계층화: 계층 구조가 명확한가?
3. 확장성: 변경에 유연한가?
4. 패턴 활용: 디자인 패턴이 적절히 활용되었는가?

출력 형식:
- 모듈화: X/5 - [근거]
- 계층화: X/5 - [근거]
- 확장성: X/5 - [근거]
- 패턴 활용: X/5 - [근거]
```

### 4. 서사적 일관성 분석

**프로젝트 스토리라인 추적:**
```
다음 3단계 프로젝트의 서사적 일관성을 분석하세요:

1. 문제 정의: 01-plan에서 정의한 문제가 일관된가?
2. 해결책: 제시한 해결책이 문제와 연결되는가?
3. 결과: 결과가 해결책의 효과를 보여주는가?
4. 기여도: 연구의 기여도가 명확히 드러나는가?

출력 형식:
- 문제 정의: X/5 - [근거]
- 해결책 연결: X/5 - [근거]
- 결과 연결: X/5 - [근거]
- 기여도 표현: X/5 - [근거]
```

## 점수화 체계

각 하위 항목은 1-5점으로 평가됩니다:

- **5점 (우수):** 해당 기준이 탁월하게 충족됨, 모범 사례 수준
- **4점 (양호):** 해당 기준이 잘 충족됨, 약간의 개선 가능
- **3점 (보통):** 해당 기준이 충족됨, 개선이 필요함
- **2점 (미흡):** 해당 기준이 부분적으로 충족됨, 상당한 개선 필요
- **1점 (부적합):** 해당 기준이 충족되지 않음, 재작업 필요

## 사용자 피드백 통합

AI 평가 결과는 사용자에게 제시되고, 사용자는 다음을 할 수 있습니다:

1. **점수 조정:** 근거를 보고 점수를 조정
2. **추가 근거 제시:** AI가 놓친 근거를 추가
3. **재평가 요청:** 수정 후 재평가 요청

이 과정을 통해 AI 분석과 사용자 판단이 결합된 더 정확한 평가가 가능합니다.
