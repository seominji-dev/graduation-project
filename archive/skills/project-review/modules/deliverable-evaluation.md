# Deliverable Evaluation Module

산출물 완성도 평가 모듈 - 논문, 발표, 데모의 품질을 평가합니다.

## 평가 차원: 산출물 완성도 (25점)

### 1. 논문 (10점)

**학술적 서지 (5점):**
- **논리 구조 (Logical Structure):** 논리 구조가 타당한가?
  - 도입-본론-결론의 구조가 명확한가?
  - 각 섹션간 연결이 자연스러운가?

- **인용 (Citation):** 인용이 적절하고 일관된가?
  - 필요한 곳에 인용이 있는가?
  - 인용 스타일이 일관된가?

- **학술적 용어 (Terminology):** 학술적 용어 사용이 적절한가?
  - 전문 용어를 정확히 사용하는가?
  - 약어가 적절히 정의되었는가?

**시각 자료 (3점):**
- **표/그래프 (Tables/Graphs):** 표와 그래프가 적절한가?
  - 필요한 곳에 포함되었는가?
  - 명확하고 이해하기 쉬운가?
  - 캡션과 설명이 충분한가?

**문서화 (2점):**
- **형식 (Format):** 논문 형식을 따르는가?
  - 페이지 번호, 머리말 등이 적절한가?
  - 폰트, 줄 간격 등이 적절한가?

### 2. 발표 자료 (10점)

**시각적 계층 구조 (4점):**
- **핵심 메시지 (Key Message):** 슬라이드당 핵심 메시지가 명확한가?
  - 한 슬라이드에 하나의 핵심 메시지인가?
  - 메시지가 즉시 파악되는가?

- **정보 우선순위 (Visual Priority):** 정보의 시각적 우선순위가 적절한가?
  - 중요한 정보가 눈에 띄는가?
  - 불필요한 정보가 없는가?

- **여백과 레이아웃 (Whitespace & Layout):** 여백과 레이아웃이 균형 잡혀 있는가?
  - 적절한 여백이 있는가?
  - 요소들이 정돈되어 있는가?

**스토리텔링 (4점):**
- **서사 구조 (Narrative Structure):** 도입-전개-결론의 구조가 명확한가?
  - 문제 제시 → 해결책 → 결과의 흐름이 자연스러운가?
  - 청중을 끌어들이는 도입인가?

- **호기심 유발 (Curiosity):** 청중의 호기심을 유발하는가?
  - 왜 이 프로젝트가 중요한지 전달되는가?
  - 흥미로운 사실이나 질문으로 시작하는가?

- **기술적 명확성 (Technical Clarity):** 기술적 내용이 명확하게 전달되는가?
  - 전문 용어가 적절히 설명되는가?
  - 복잡한 개념이 쉽게 설명되는가?

**시각 자료 품질 (2점):**
- **다이어그램 (Diagrams):** 다이어그램이 명확하고 이해하기 쉬운가?
  - 아키텍처 다이어그램이 명확한가?
  - 흐름도가 이해하기 쉬운가?

- **그래프/차트 (Charts):** 그래프/차트가 적절한 형식인가?
  - 데이터 시각화가 적절한 형식인가?
  - 축, 범례가 명확한가?

### 3. 데모 (5점)

**기능 시연 (2점):**
- **핵심 기능 (Core Features):** 핵심 기능이 모두 시연되는가?
  - 계획된 모든 기능을 보여주는가?
  - 실제 사용 시나리오를 보여주는가?

**사용자 경험 (2점):**
- **UI/UX (User Experience):** UI/UX가 직관적인가?
  - 인터페이스가 사용하기 쉬운가?
  - 에러 처리가 우아한가?

**기술적 표현 (1점):**
- **난이도 표현 (Complexity):** 구현의 어려움이 잘 전달되는가?
  - 기술적 난이도가 적절히 표현되는가?
  - 독창적인 해결책이 강조되는가?

## AI 분석 프롬프트

**논문 평가:**
```
다음 논문을 평가하세요:

1. 논리 구조를 분석하세요.
2. 인용의 적절성과 일관성을 확인하세요.
3. 표/그래프의 품질을 평가하세요.
4. 학술적 서지 전반을 평가하세요.

각 항목에 1-5점을 부과하고 구체적 피드백을 제시하세요.
```

**발표 자료 평가:**
```
다음 발표 자료를 평가하세요:

1. 시각적 계층 구조를 분석하세요.
2. 스토리텔링 흐름을 평가하세요.
3. 시각 자료(다이어그램, 차트)의 품질을 평가하세요.

각 항목에 1-5점을 부과하고 구체적 피드백을 제시하세요.
```

**데모 평가:**
```
다음 데모 설명/영상/스크린샷을 평가하세요:

1. 핵심 기능 시연을 확인하세요.
2. UI/UX 적절성을 평가하세요.
3. 기술적 난이도 표현을 확인하세요.

각 항목에 1-5점을 부과하고 구체적 피드백을 제시하세요.
```
