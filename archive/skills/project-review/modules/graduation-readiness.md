# Graduation Readiness Module

졸업 적합성 평가 모듈 - 졸업 작품으로서의 적합성을 평가합니다.

## 평가 차원: 졸업 적합성 (10점)

### 1. 학위 수준 적합성 (5점)

**평가 기준:**

**학사 수준 (Bachelor's):**
- **문제 정의:** 명확하고 잘 정의된 문제
- **해결책:** 기존 방법을 적용하거나 적절한 수정
- **구현:** 작동하는 프로토타입 또는 시스템
- **보고:** 논리적이고 명확한 보고서
- **기여도:** 실용적 가치 또는 학습 가치

**석사 수준 (Master's):**
- **문제 정의:** 연구 간극을 식별하는 명확한 문제
- **해결책:** 새로운 방법 제안 또는 기존 방법의 새로운 응용
- **구현:** 완전히 작동하는 시스템 또는 포괄적인 연구
- **보고:** 학술적으로 수준 높은 보고서
- **기여도:** 학술적 또는 실용적 기여

**평가 항목:**
- **난이도 (Difficulty):** 해당 학위 수준에 부합하는 난이도인가?
  - 문제의 복잡성이 적절한가?
  - 해결책의 독창성이 수준에 맞는가?

- **깊이 (Depth):** 해당 학위 수준에 맞는 깊이인가?
  - 문헌 조사가 충분한가?
  - 기술적/이론적 깊이가 충분한가?

- **범위 (Scope):** 해당 학위 수준에 맞는 범위인가?
  - 너무 좁지도 너무 넓지도 않은가?
  - 주어진 시간에 완료 가능한 범위인가?

**AI 분석 프롬프트:**
```
다음 졸업 프로젝트의 학위 수준 적합성을 평가하세요:

1. 프로젝트의 문제 정의를 확인하세요.
2. 해결책의 독창성과 난이도를 평가하세요.
3. 문헌 조사와 기술적 깊이를 확인하세요.
4. 해당 프로젝트가 학사/석사 어느 수준에 더 가까운지 판단하세요.

학사/석사 각각의 기준에 부합하는지 1-5점으로 평가하세요.
```

### 2. 독창성 및 기여도 (5점)

**평가 항목:**

**독창성 (Originality) - 3점:**
- **새로운 방법 (New Method):** 새로운 방법론을 제안하는가?
  - 기존에 없는 방법인가?
  - 기존 방법의 새로운 조합인가?

- **새로운 응용 (New Application):** 기존 방법을 새로운 문제에 적용하는가?
  - 기존 방법을 새로운 도메인에 적용하는가?
  - 새로운 응용 분야를 식별하는가?

- **개선 (Improvement):** 기존 방법을 개선하는가?
  - 성능, 효율성, 정확도를 개선하는가?
  - 개선의 근거가 명확한가?

**기여도 (Contribution) - 2점:**
- **학술적 기여 (Academic Contribution):** 학술적 기여가 있는가?
  - 새로운 지식을 추가하는가?
  - 연구 간극을 메우는가?

- **실용적 기여 (Practical Contribution):** 실용적 기여가 있는가?
  - 실제 문제를 해결하는가?
  - 산업/사회에 활용 가능한가?

**AI 분석 프롬프트:**
```
다음 졸업 프로젝트의 독창성과 기여도를 평가하세요:

1. 프로젝트의 독창적인 요소를 식별하세요.
2. 새로운 방법, 응용, 개선 중 어디에 해당하는지 확인하세요.
3. 학술적/실용적 기여도를 평가하세요.

각 항목에 1-5점을 부과하고 근거를 제시하세요.
```

## 졸업 적합성 판정 기준

| 점수 | 등급 | 판정 | 의미 |
|------|------|------|------|
| 9-10 | A | Ready (적합) | 졸업 작품으로 완벽 |
| 7-8 | B | Conditional (조건부) | 약간의 개선으로 적합 |
| 5-6 | C | Review Needed (검토 필요) | 주요 개선 필요 |
| 0-4 | D/F | Not Ready (부적합) | 상당한 재작업 필요 |

## 출력 형식

```yaml
graduation_readiness:
  degree_appropriateness:
    difficulty: X/2
    depth: X/2
    scope: X/1
    total: X/5
    target_degree: "[Bachelor's|Master's]"
    feedback: "[구체적 피드백]"

  originality_contribution:
    originality:
      new_method: X/1
      new_application: X/1
      improvement: X/1
      total: X/3
    contribution:
      academic: X/1
      practical: X/1
      total: X/2
    overall_total: X/5
    feedback: "[구체적 피드백]"

  overall_score: X/10
  grade: "[A-F]"
  readiness: "[Ready|Conditional|Review Needed|Not Ready]"
  recommendation: "[구체적 권장사항]"
```

## 졸업 요건 체크리스트

**필수 항목 (Must Have):**
- [ ] 명확한 연구 질의(RQ)
- [ ] 문헌 조사 섹션
- [ ] 방법론 섹션
- [ ] 결과 및 결론 섹션
- [ ] 작동하는 코드/시스템
- [ ] 논문
- [ ] 발표 자료
- [ ] 데모

**권장 항목 (Should Have):**
- [ ] 한계 및 future work
- [ ] 성능 평가/벤치마크
- [ ] 사용자 매뉴얼
- [ ] API 문서
- [ ] 테스트 스위트

**선택 항목 (Nice to Have):**
- [ ] GitHub Pages/웹사이트
- [ ] 동영상 데모
- [ ] 포스터
- [ ] 기술 블로그 포스트
