### 1단계 (오늘 ~ 1시간 안에)

1. **이 README 끝까지 읽기** ← 지금 하고 있음

3. **`numbers-cheatsheet.md`의 "절대 틀리면 안 되는 5개 숫자" 외우기** (10분)

1. **`glossary.md` 훑어보기** (모르는 용어 표시)
2. **보고서 1~3장 통독** (`06-final-report/final/final-report.docx`)

2. **`self-recording-evaluation-form.md`로 자가 평가**
3. **Q&A 11문항 각 1회씩 연습**

### 4단계 (발표 3일 전 = 5/23경)

1. **풀 리허설 2~3회 녹음**
2. **백업 녹화본 제작** (`backup-demo-shooting-kit.md`)

1. **외부 모니터 연결 풀 리허설** (프로젝터 환경 시뮬레이션)

3. **장비 지참 최종 확인**

1. **아침 3분 숫자 복습** (`numbers-cheatsheet.md` 끝 부분)
2. **Ollama warmup** (`ollama run gemma4:e4b "hi"` → `/bye`)
3. **`emergency-card.md` 요약 1회**
4. **발표장 도착 후 `projector-checklist.md` 순서대로 실행**

## ❓ 자주 묻는 질문 (FAQ)

### Q1. 이 자료 전부를 읽어야 하나요?

**필수**: `rehearsal-playbook.md`, `numbers-cheatsheet.md`, `glossary.md`, `slide-cards-12.md`, `qa-student-voice.md`

**선택**: `unexpected-questions.md` (심화), `supervisor-checklist.md` (도와주는 사람이 읽을 것)

**당일 아침**: `emergency-card.md`, `projector-checklist.md`

### Q2. 시간이 부족하면?

**최소 필수 읽기 순서** (약 2시간):

1. `rehearsal-playbook.md` (30분)
2. `numbers-cheatsheet.md`의 "절대 틀리면 안 되는 5개 숫자" (10분)
3. `slide-cards-12.md` 전체 1회 (40분)
4. `qa-student-voice.md` Q1, Q2, Q5, Q10 (30분)
5. `emergency-card.md`의 "Tier 1 상황" (10분)

### Q3. 원본 script.md와 script-v2.md 중 어느 걸 봐야 하나요?

**연습 중**: `script-v2.md` (서민지 말투, 호흡 표기, 백업 한 줄 포함)
**현장 제출 보고서 검증용**: `script.md` (수치 정확, 정식 원고)

둘의 수치는 **완전히 동일**하다. 차이는 말투와 길이뿐.

### Q4. Rate Limiter는 토큰 버킷인가요, 슬라이딩 윈도우인가요?

**발표에서 답변**: "토큰 버킷 방식입니다" (보고서 정본)
**추가 설명 요청 시**: "개념은 토큰 버킷, 구현은 1분 슬라이딩 윈도우 카운터로 단순화했습니다."

자세한 내용은 `glossary.md`의 "D. 시스템·프로토콜 용어" 섹션 참조.

### Q5. Ollama 모델은 뭐였죠?

**`gemma4:e4b`** (Google Gemma 4 계열, 약 40억 파라미터)

- 설치: `ollama pull gemma4:e4b`
- warmup: `ollama run gemma4:e4b "hi"` → `/bye`

## 🎤 발표 당일 마지막 한 마디

이 자료들은 서민지의 **실수 가능성을 줄이기 위한 안전망**이다. 발표 당일 자신감을 잃었다면 다음을 기억하자.

- **충분히 준비했다** — 이 자료를 다 봤다면 정말 충분하다.
- **실수해도 괜찮다** — `emergency-card.md`에 모든 대응책이 있다.
- **네 연구는 진짜다** — 네가 직접 구현하고, 실험하고, 분석한 결과다.
- **청중은 네 편이다** — 발표를 통해 네가 한 일을 공유하러 온 것.

행운을 빈다.
