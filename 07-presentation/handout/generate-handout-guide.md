# 핸드아웃(현장 제출용) 생성 가이드

> 이 문서의 목적: **발표장에 제출할 슬라이드 종이 인쇄물**을 만드는 방법을 정리한다.
> 참고: `07-presentation/slides/presentation.pptx` 원본에서 2-in-1 또는 4-in-1 배치로 PDF를 만들어 인쇄한다. 컬러 프린터 사용 권장.

---

## 🎯 제출 요건

- **형식**: A4 컬러 인쇄
- **배치**: 슬라이드 2-in-1 또는 4-in-1 (상황에 따라 선택)
- **부수**: 최소 3부 (본인용 1 + 제출용 1 + 여분 1)
- **언제 준비**: 발표 **3일 전까지**
- **언제 인쇄**: 발표 **1일 전 또는 당일 아침** (가장 최신 슬라이드 반영 위해)

---

## 📋 두 가지 배치 방식 비교

### 2-in-1 배치 (6페이지)

- A4 한 면에 슬라이드 2장 배치
- 총 **12 슬라이드 ÷ 2 = 6페이지**
- **가독성 ★★★** (글자 충분히 크다)
- **권장 상황**: 교수님·청중이 발표 중 참고할 예정일 때

### 4-in-1 배치 (3페이지)

- A4 한 면에 슬라이드 4장 배치
- 총 **12 슬라이드 ÷ 4 = 3페이지**
- **가독성 ★★** (표·숫자가 작게 보임)
- **권장 상황**: 단순 종이 제출용, 발표 중 참고 목적이 아닐 때

### 결정 권장

**기본값**: **2-in-1** 추천. 가독성 우선.
**예외**: 종이 절약이 중요하거나 제출 수량이 많을 때만 4-in-1.

## 🛠️ macOS에서 PDF 생성하는 방법

### 방법 A — PowerPoint/Keynote 내장 인쇄 대화상자 (가장 간단)

#### PowerPoint (Microsoft 365)

1. `07-presentation/slides/presentation.pptx` 열기
2. **파일 → 인쇄** (단축키: `Cmd + P`)
3. **Printer** 드롭다운 → **"PDF로 저장"** 선택 (하단 PDF 메뉴)
4. **Layout** 설정:
   - 2-in-1: "한 페이지에 2개 슬라이드"
   - 4-in-1: "한 페이지에 4개 슬라이드"
5. **Slide Size**: "Handouts 2 slides per page" 또는 "Handouts 4 slides per page"
6. **Frame Slides** 체크 (슬라이드 외곽에 얇은 테두리)
7. **저장 위치**: `07-presentation/handout/presentation-handout-2in1.pdf`

#### Keynote (Apple)

1. `.pptx`를 Keynote에서 열기 (자동 변환)
2. **파일 → 프린트** (`Cmd + P`)
3. **인쇄 레이아웃**: "유인물"
4. **페이지당 슬라이드**: 2 또는 4
5. **대비 향상** 체크 (프로젝터와 다른 색으로 인쇄될 경우 대비)
6. **PDF로 저장**

### 방법 B — LibreOffice Impress (무료 대안)

1. LibreOffice Impress에서 `.pptx` 열기
2. **파일 → 내보내기 → PDF로 내보내기**
3. **일반 탭**:
   - **표지 페이지** 사용 안 함
   - **범위**: 전체
4. **PDF 옵션**에서 유인물 2-in-1 설정 (버전에 따라 메뉴 다름)

### 방법 C — 명령줄 (고급)

```bash
# pptx → PDF 변환 (LibreOffice 필요)
soffice --headless --convert-to pdf \
  /Users/truestone/Dropbox/repo/mj/졸업프로젝트/07-presentation/slides/presentation.pptx \
  --outdir /Users/truestone/Dropbox/repo/mj/졸업프로젝트/07-presentation/handout/

# 2-in-1 레이아웃 변환 (PDFjam 필요, 선택)
pdfjam --nup 1x2 --landscape --a4paper presentation.pdf \
  --outfile presentation-handout-2in1.pdf
```

## 🖨️ 인쇄 설정

### 용지

- **A4 (210 × 297 mm)**
- **컬러 인쇄** (흑백 인쇄 시 그래프 구분 어려움)

### 양면/단면

- **단면 인쇄** 권장 (양면은 뒷면 비치는 문제 있을 수 있음)

### 여백

- **표준 여백** (상하좌우 각 20mm 정도)

### 화질

- **Fine 또는 Best** 설정 (사진 인쇄 수준)

### 인쇄소 이용 시

**학교 근처 인쇄소**:
- "A4 컬러 양면 인쇄" 요청 (단면도 좋음)
- 슬라이드 12장을 2-in-1으로 6장 인쇄 (한 부에 6장)
- 비용: 장당 100~300원 (컬러 기준)
- **소요 시간**: 10분 정도 (여유롭게 30분 전에 방문)

**편의점 프린터 (세븐일레븐, CU 등)**:
- USB에 PDF 저장해서 가져가기
- 컬러 A4 인쇄 가능, 장당 약 500원
- **주의**: 파일 형식 제한 있을 수 있음 (PDF가 가장 안전)

## 🔍 인쇄 전 품질 체크

### PDF 파일 열어서 확인

- [ ] 표지 슬라이드의 학과·이름·주제가 선명
- [ ] 슬라이드 9(WFQ 차등) 표의 4개 숫자가 읽힘 (1,862 / 8,384 / 15,109 / 22,029)
- [ ] 슬라이드 5(알고리즘 비교)의 4개 박스가 구분됨
- [ ] 그래프 색상이 구분됨 (등급별 4색)
- [ ] 페이지 번호가 표시됨 (선택)

### 인쇄 후 확인

- [ ] 종이 색상과 비교해 대비가 충분한가
- [ ] 글자가 뭉개지지 않았나
- [ ] 표의 격자선이 또렷한가
- [ ] 그림 캡션이 가독되나

## 📎 제출 시 마무리

### 스테이플러 처리

- 좌측 상단 **한 번 찍기** (해체하기 쉬움)
- 스테이플러 없으면 **클립** 사용

### 라벨 추가 (선택)

발표장에 여러 제출물이 쌓이므로, 누구 것인지 구분하는 라벨을 첫 페이지에 붙이면 좋다.

**라벨 내용 예시**:
```
졸업 프로젝트 최종 발표
홍익대학교 컴퓨터공학과 4학년 서민지
주제: 스케줄링 알고리즘을 활용한 다중 사용자 LLM API 요청 관리 시스템
발표일: 2026년 5월 ○일
```

## 📦 지참 수량

- **본인용**: 1부 (발표 중 참고, 담당자용)
- **제출용**: 1부 (현장 제출)
- **여분**: 1~2부 (교수님·조교가 추가 요청할 수 있음)
- **총계**: **3~4부**

## 🔄 재인쇄 트리거

**다음 경우에는 기존 인쇄물 폐기, 재인쇄**:

- 슬라이드 내용이 변경된 경우 (재확인 필수)
- 숫자가 업데이트된 경우 (예: "약 12배"로 정정)
- 그래프·그림이 변경된 경우

발표 **1주 전부터는 슬라이드 변경 금지**가 원칙. 마지막 주에는 인쇄물 최종본을 확정.
