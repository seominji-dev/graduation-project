# 지도 담당자용 점검 체크리스트

> 이 문서의 목적: **지도교수·선배·보호자 등 서민지의 발표 준비를 도와주는 사람**이 단계별로 상태를 확인하는 체크리스트.
> 사용 방법: 각 단계에서 서민지와 함께 확인. 모든 체크가 완료되면 발표 준비 완료로 판단.

---

## 🎯 지도 담당자의 역할

서민지 본인이 놓칠 수 있는 부분을 **외부 관점**에서 확인하는 것이 핵심.

특히 주의할 영역:
- **시간 관리** (본인은 주관적으로 판단하기 어려움)
- **발표 내용의 정확성** (수치 오류·용어 혼동)
- **발표 태도** (목소리 크기·속도·자세)
- **기술적 환경** (노트북·Ollama·데모)
- **심리적 준비** (긴장 상태 관찰)

---

## 📋 단계별 체크리스트

## Phase A — 1주 전 (5/19경) 점검

### A1. 자료 숙지도 확인

- [ ] 서민지가 **보고서 1~7장을 1회 이상 통독**했는가
- [ ] **핵심 숫자 6개**를 즉답할 수 있는가
  - 예: "WFQ에서 Enterprise가 Free보다 몇 배 빠른가요?" → "약 12배"
  - 예: "Ollama 모델 이름이 뭔가요?" → "gemma4:e4b"
- [ ] **Q&A 11문항 중 3문항을 무작위로 질문했을 때** 주저 없이 답변하는가
- [ ] **연구 질문 3개 (RQ1/RQ2/RQ3)**를 순서대로 말할 수 있는가

### A2. 기술 환경 확인

- [ ] **노트북에 Ollama가 설치**되어 있고 `ollama --version`이 응답하는가
- [ ] **gemma4:e4b 모델이 다운로드**되어 있는가 (`ollama list` 확인)
- [ ] `08-final-submission/source-code/`에서 **`npm install` 완료** 상태인가
- [ ] **서버 기동 (`npm start`)**이 에러 없이 되는가
- [ ] **데모 시나리오 1**이 문서대로 실행되는가 (시연 확인)

### A3. 1차 리허설 관찰

- 서민지가 스마트폰 녹음과 함께 1차 리허설 실시
- 지도자가 옆에서 들으며 아래 항목 체크

- [ ] **전체 시간**이 10~14분 범위인가
- [ ] **슬라이드별 시간 배분**이 적절한가 (한 슬라이드에 너무 오래 머물지 않는가)
- [ ] **말의 속도**가 적당한가 (너무 빠르면 긴장, 너무 느리면 지루함)
- [ ] **목소리 크기**가 일정한가 (뒷자리 청중까지 들리는 수준)
- [ ] **숫자 발음**이 또렷한가 (특히 "만 일천 팔백"류)
- [ ] **막힌 구간**이 5곳 이하인가

### A4. 개선 방향 조언

1차 리허설 후 서민지와 함께:
- 가장 자연스러웠던 구간 1개 칭찬
- 가장 개선 필요한 구간 1~2개 지적
- 3일 전까지 할 구체 과제 제시

## Phase B — 3일 전 (5/23경) 점검

### B1. Q&A 답변 숙지도 확인

무작위로 Q&A 11문항 중 **5개를 골라** 질문:
- [ ] Q2 (JFI 정의)에 대한 답변이 30초 이내이며 정확한가
- [ ] Q3 (MLFQ 선점형 실서버 불가)의 이유를 정확히 설명하는가
- [ ] Q4 (WFQ 가중치 결정)에서 "100:50:10:1"을 말하는가
- [ ] Q9 (VFT 비용=1 고정)의 이유를 정확히 설명하는가
- [ ] Q10 (연구 의의)에 대한 답변이 자신감 있게 나오는가

### B2. 데모 실행 확인

- [ ] **시나리오 1** 실행 (FCFS→WFQ 순서 비교): 3분 이내 완료
- [ ] **시나리오 2** 실행 (Rate Limiter 429): 1분 이내 완료
- [ ] **시나리오 3** 실행 (Ollama 실호출 축소): 3분 이내 완료
- [ ] 데모 중 서민지가 **구두 설명을 병행**하는가

### B3. 백업 준비 확인

- [ ] **백업 녹화본 `backup-demo.mov`** 파일 존재
- [ ] 녹화본 재생 → **화질·가독성 확인** (글자 보이는가)
- [ ] 녹화본이 **USB 메모리에도 백업**되어 있는가

### B4. 2차 리허설 관찰 (필수)

1차 대비 개선된 부분이 있는지 확인:
- [ ] 1차에서 지적한 **막힌 구간이 해결**됐는가
- [ ] **시간 관리**가 개선됐는가 (표준편차 축소)
- [ ] **자신감**이 느껴지는가 (목소리 떨림 감소)

### B5. 심리 상태 확인

- 서민지와 대화하며 관찰:
- [ ] 과도한 불안·긴장이 보이는가
- [ ] 스트레스로 인한 식사·수면 문제가 있는가
- [ ] 완벽주의로 무한 수정을 하고 있지는 않은가

**만약 B5 문제 발견 시**:
- 충분한 휴식 권유
- "지금 수준으로 충분하다"는 안정화 메시지
- 필요하면 리허설 횟수 줄이고 휴식

## Phase C — 1일 전 (5/25경) 점검

### C1. 최종 리허설 관찰 (외부 모니터 환경)

- 노트북을 외부 모니터/TV에 HDMI로 연결한 상태에서 풀 리허설
- [ ] **슬라이드가 외부 모니터에 정상 표시**되는가
- [ ] **터미널·브라우저 전환**이 부드럽게 되는가
- [ ] **시간 목표 달성**: 슬라이드 1~11 = 10분 20초 이내
- [ ] **데모 시나리오 1, 2 연속 실행** = 3분 30초 이내
- [ ] Q&A 응답 시 **보고서 절 번호**를 언급하는가

### C2. 지참물 최종 확인

함께 가방을 열어 확인:
- [ ] 노트북 + 충전기 (100W 이상)
- [ ] HDMI 케이블 + USB-C 어댑터
- [ ] 마우스 (선택)
- [ ] 프레젠터 리모컨 (선택)
- [ ] USB 메모리 (백업 녹화본 + 소스코드 사본)
- [ ] 학생증, 필기구
- [ ] **인쇄본 3~4부** (2-in-1 컬러 인쇄)

### C3. 당일 일정 확정

- [ ] **발표 시간** (몇 시 몇 분부터)
- [ ] **발표장 위치**
- [ ] **도착 시간** (최소 30분 전)
- [ ] **교통수단** (대중교통 / 자가용)

### C4. 심리적 안정화

- [ ] "충분히 준비됐다"고 말해주기
- [ ] 완벽보다는 **완수가 중요**하다는 메시지 전달
- [ ] **22시 이전 취침** 권장

## Phase D — 발표 당일 (5/26~29 중) 점검

### D1. 아침 체크 (발표 3시간 전)

서민지와 대화하며:
- [ ] 잠을 충분히 잤는가
- [ ] 아침 식사를 했는가 (과식은 금물)
- [ ] 긴장도는 어떤가 (너무 높지 않은가)
- [ ] 지참물은 다 챙겼는가

### D2. 현장 도착 시 (발표 20분 전)

서민지 옆에서 지원 (가능한 경우):
- [ ] 프로젝터 연결·테스트 지원
- [ ] Ollama 서버 기동 확인
- [ ] 슬라이드 표시 상태 확인

### D3. 발표 시작 전 (발표 5분 전)

- [ ] "차분히, 천천히 시작해라" 조언
- [ ] "실수해도 괜찮다" 메시지
- [ ] 물병 준비 확인

### 점수 기준

- **22점 이상**: 완벽 준비. 당일 평소대로 하면 됨.
- **18~21점**: 준비 양호. 작은 보완만.
- **14~17점**: 준비 부족. 추가 리허설 필요.
- **13점 이하**: 기본 준비가 안 됨. 일정 재조정 필요.

## 🎯 특별 관찰 포인트

### 서민지가 자주 놓치는 것 (본인이 인식하지 못할 가능성이 높음)

1. **목소리 크기 저하**: 슬라이드 9 이후로 목소리가 작아지는 경향
2. **숫자 실수**: "1,862"와 "1,826" 같은 혼동
3. **너무 빠른 속도**: 긴장하면 10~20% 빨라짐
4. **전환 멘트 생략**: "(전환)" 문장을 빼먹는 경우
5. **Q&A에서 말끝 흐림**: 답변 마지막 부분을 작게 말함

→ 이 중 1~2개만 지적해 개선하면 전체 완성도 상승.

### 서민지의 강점 (이미 잘 하는 것)

- 연구 내용의 논리 구성
- 숫자의 정확성
- 보고서·슬라이드의 완성도

→ **과도한 개입 지양**. 이미 잘하는 부분은 유지.

## 🎬 리허설 시 지도자 피드백 템플릿

### 긍정 피드백 (먼저 하기)

- "슬라이드 X에서 ~한 설명이 정말 명확했어."
- "Q&A에서 ~한 답변이 자신 있게 나오네."
- "전체 흐름이 자연스럽다."

### 개선 피드백 (1~2개만)

- "슬라이드 Y에서 [구체 내용]을 이렇게 말하면 더 좋겠어."
- "숫자 Z를 말할 때 조금 더 천천히."
- "Q&A 답변 끝에 '이상입니다' 붙이면 더 깔끔할 것 같아."

### 금지 피드백

- "~는 왜 그렇게 했어?" (공격적)
- "내가 해봤더라면 ~" (비교)
- "다들 ~하던데" (일반화)
- 10가지 이상의 수정 요구 (과부하)

## 📝 지도자의 자기 점검

내가 서민지를 제대로 지도하고 있는가?

- [ ] 서민지의 **자신감을 깎지 않는 방식**으로 피드백하는가
- [ ] **너무 많은 수정 요구**를 하지 않는가 (한 번에 1~2개만)
- [ ] **서민지의 고유한 강점**을 존중하는가
- [ ] **기술적으로 정확한** 조언만 하는가 (추측 금지)
- [ ] **감정적 지원**도 함께 제공하는가
