# 현장 응급 카드 — 무엇이 잘못돼도 당황하지 않는다

> 이 문서의 목적: 발표 현장에서 **예상치 못한 일이 생겨도 침착하게 대응**할 수 있도록 상황별 대본과 절차를 한 장에 모은 것.
> 사용 방법: 발표 직전 이 카드 한 장만 출력해서 노트북 옆에 두면 된다. 눈으로 한 번만 훑어도 된다.

---

## 🚨 Tier 1 — 치명적 상황 (데모 완전 실패)

### 상황 1-A: 서버가 기동 안 됨 (`npm start` 실패)

**증상**: 터미널에서 에러가 떠서 `http://localhost:3000`이 안 열림.

**30초 안에 할 것**:
1. **침착한 멘트**: "시스템에 작은 이슈가 생겼습니다. 미리 준비한 녹화본으로 같은 내용을 보여드리겠습니다."
2. **백업 녹화본 재생**: `07-presentation/demo/backup-demo.mov` (미리 시연 완료 상태)
3. **녹화본 중간 멘트**: "이 구간은 FCFS가 도착 순서대로 처리하는 모습입니다" 등 자막 역할 수행
4. **녹화본 종료 후**: "이렇게 실제 시스템에서도 시뮬레이션과 같은 경향이 확인됐습니다. 그럼 결론으로 돌아가겠습니다."

**절대 하지 말 것**: 에러 메시지를 청중에게 보여주며 "뭐가 안 되는지 모르겠네요..." 하는 모습.

### 상황 1-B: Ollama 응답 없음 (시나리오 1-7, 3 관련)

**증상**: Ollama 서버가 멈췄거나 응답이 20초 이상 안 옴.

**30초 안에 할 것**:
1. **즉시 전환**: "실제 LLM 호출은 로컬 환경 이슈로 다음에 보여드리겠습니다. 대신 시뮬레이션 데모로 같은 로직을 확인해 드리겠습니다."
2. **시나리오 1-1 ~ 1-5** (Ollama 없이 동작하는 부분)만 실행
3. **구두 설명 보완**: "실서버 실험 결과는 보고서 5.4절과 슬라이드 10에서 확인하실 수 있습니다. 평균 처리 시간이 208ms로 세 알고리즘이 거의 동일했다는 점이 핵심입니다."

### 상황 1-C: 프로젝터 화면이 노트북과 불일치 또는 검은 화면

**증상**: 프로젝터에 아무것도 안 나오거나 노트북 화면과 다른 해상도.

**1분 안에 할 것**:
1. **침착한 멘트**: "디스플레이 설정을 잠깐 조정하겠습니다. 잠시만 기다려 주세요."
2. **System Settings → Displays → Arrange → Mirror Displays** 체크/해제 토글
3. **여전히 안 되면**: HDMI 케이블 재연결 (빼고 3초 후 다시 꽂기)
4. **그래도 안 되면**: "PPT를 출력물로 준비했으니, 음성 발표로 먼저 진행하고 필요시 인쇄물을 참고해 주시면 감사하겠습니다."

## 🟡 Tier 2 — 발표 흐름 위기 (일시 정지 필요)

### 상황 2-A: 완전히 말문이 막힘

**증상**: 슬라이드를 봤는데 아무 말도 떠오르지 않음.

**10초 안에 할 것**:
1. **물 한 모금 마시기** (자연스러운 휴식)
2. **각 슬라이드의 "막혔을 때 백업 한 줄" 사용** (→ `slide-cards-12.md` 참조)
3. **자연스러운 전환 멘트**: "이 부분은 슬라이드에 정리되어 있어서 그대로 참고하시면 됩니다. 다음 슬라이드로 넘어가겠습니다."

**절대 금지**: "어... 음... 뭐였더라..." 10초 이상 정적.

### 상황 2-B: 숫자를 잘못 말함

**증상**: "약 15배"라고 말했는데 정본은 "약 12배"임.

**5초 안에 할 것**:
1. **즉시 정정**: "죄송합니다, 약 12배입니다. 정확한 수치는 슬라이드에 나와 있습니다."
2. **멈추지 말고 계속 진행**: 청중은 숫자 실수에 크게 신경 쓰지 않는다. 정정한 사실 자체로 오히려 신뢰감 상승.

**참고**: 치트시트에 "약 12배"가 최우선 숫자. → `numbers-cheatsheet.md`

### 상황 2-C: 시간 초과 (8분 지점에 슬라이드 8 미도달)

**증상**: 8분 지났는데 아직 실험 2 슬라이드.

**즉시 할 것**:
1. **슬라이드 7, 8, 9를 묶어서 간략 발표**: 각 1분 → 각 30초로 단축
2. **슬라이드 8의 핵심만**: "MLFQ 선점형은 짧은 요청 73% 감소, 긴 요청 89% 증가, 실서버 적용 불가"
3. **슬라이드 9의 핵심만**: "Enterprise가 Free보다 약 12배 빨랐다"
4. **데모는 시나리오 1만 실행**: 시나리오 2는 Q&A에서 요청 시 보여드림

**참고**: 시간 체크포인트 → `script-v2.md` 또는 `demo-scenario.md`

### 상황 2-D: 청중이 질문을 중간에 함

**상황**: 슬라이드 5에서 교수님이 "MLFQ 부스팅은 언제 적용되나요?"라고 끼어드심.

**즉시 할 것**:
1. **정중히 대응**: "네, 좋은 질문이십니다. 간략히 말씀드리면 저희는 5초 주기로 부스팅을 적용했습니다."
2. **흐름 복귀**: "상세한 내용은 슬라이드 8에서 다루고 있으니, 거기서 다시 보여드리겠습니다. 지금은 슬라이드 5를 마저 설명드리겠습니다."

**포인트**: 질문을 무시하지 않되, 발표 흐름으로 자연스럽게 복귀.

## 🟢 Tier 3 — 경미한 이슈 (무시 가능)

### 상황 3-A: 말 실수 (단어 한두 개)

**증상**: "스케줄링"을 "스케줄러링"으로 말함.

**대응**: 무시하고 계속 진행. 청중은 알아채지 못한다.

### 상황 3-B: 기침이 나옴

**대응**: "죄송합니다" 한 마디 하고 물 한 모금. 그리고 계속 진행.

### 상황 3-C: 발표 중 알림음 울림

**대응**: "죄송합니다" 한 마디 후 노트북 음소거 확인 (이미 발표 전에 해둬야 할 것).

### 상황 3-D: 마이크 문제

**대응**: 마이크가 있으면 손으로 가볍게 톡톡 쳐서 테스트. 계속 문제 있으면 "마이크 없이 진행해도 될까요?" 문의.

## 🔵 Q&A 중 돌발 상황

### 상황 Q-A: 질문이 무슨 뜻인지 모르겠음

**대응 템플릿**:
> "질문의 요지를 한 번 더 확인하고 싶습니다. ~에 대한 질문 맞으신가요?"

→ 질문자가 다시 설명하면 답변. 그래도 모르겠으면 "죄송합니다, 그 부분은 본 연구의 범위를 벗어나서 답변드리기 어렵습니다."

### 상황 Q-B: 여러 개 질문을 동시에 받음

**대응 템플릿**:
> "세 가지 말씀해 주셨는데, 먼저 첫 번째 [주제]부터 답변드리겠습니다."

→ 하나씩 답변. 끝나면 "두 번째는..." 식으로.

### 상황 Q-C: 질문에 비판/지적이 섞여 있음

**대응 원칙**:
- **방어적 대응 금지**: "그건 오해하신 거예요" ❌
- **수용적 대응**: "네, 좋은 지적이십니다. 저희 연구의 한계 부분이기도 합니다."
- **필요시 Q&A 학부생판의 한계 답변 (Q7, Q8 등) 활용**

### 상황 Q-D: 교수님이 "실제로 보여줄 수 있나?" 요청

**대응**:
- 시나리오 3 (Ollama 실서버 축소 재현, 2~3분) 실행
- 미리 준비된 명령어 시퀀스를 따라 실행
- → `demo-scenario.md` 시나리오 3 참조

## 🔴 발표 자체가 위험해지는 극단 상황

### 상황 X-A: 노트북 배터리 급감

**증상**: 발표 중 배터리 경고.

**대응**:
1. **즉시 충전기 확인**: 콘센트 연결 상태 점검
2. **콘센트 없으면**: "잠깐 설정을 바꾸겠습니다" 하고 Low Power Mode 켜기 (약간의 시간 벌기)
3. **최악의 경우**: "배터리 이슈로 잠시 중단됩니다. 핸드아웃을 참고해 주시면..."

→ **방지책**: 발표 전 90% 이상 충전 + 충전기 지참.

### 상황 X-B: 긴장으로 어지럼증/과호흡

**대응**:
1. **물 한 모금**: 입을 적시면 심박이 안정됨
2. **심호흡 3회**: 들이쉬기 4초, 내쉬기 6초
3. **잠깐 청중 외면**: 슬라이드만 몇 초 보면서 정리
4. **여전히 힘들면**: "잠시 숨 고르겠습니다." 솔직하게 말함. 청중은 이해함.

### 상황 X-C: 목소리가 안 나옴 (성대 문제)

**대응**:
- 물 한 모금, 잠깐 침묵
- "목이 잠겨서 잠시 기다려주시면 감사하겠습니다"
- 최후의 수단: 핸드아웃 기반으로 조용히 진행 (불가피한 경우)

## 🗓️ 발표 직전 30분 안정화 루틴

### 30분 전

- [ ] 화장실 다녀오기
- [ ] 물병 준비 (500ml)
- [ ] 긴장 완화: 심호흡 10회

### 20분 전

- [ ] 노트북 설정 최종 점검:
  - 방해금지 모드 ON
  - 시스템 소리 음소거
  - 배터리 90% 이상
  - 불필요 앱 종료
- [ ] 터미널 3개, 브라우저 1개 배치 확정
- [ ] PPT 전체화면 진입 테스트 (F5)
- [ ] **Ollama warmup 1회** (`ollama run gemma4:e4b "hi"` → `/bye`)

### 10분 전

- [ ] 발표 전 **숫자 6개 암송**:
  1. "약 12배"
  2. "약 73%"
  3. "네 알고리즘: FCFS, Priority, MLFQ, WFQ"
  4. "JFI 1.000, 0.316"
  5. "Rate Limiter 100/50/10/5"
  6. "gemma4:e4b"
- [ ] 슬라이드 1~3 첫 문장 **소리 내어 1회**
- [ ] 대기 순서 확인

### 5분 전

- [ ] 청중석에서 본 프로젝터 화면 최종 확인
- [ ] 발표 시작 자세: 어깨 펴고, 미소, 청중 뒤쪽 벽 응시 준비
- [ ] "깊게 숨 한 번, 미소, 시작" 자기 암시
