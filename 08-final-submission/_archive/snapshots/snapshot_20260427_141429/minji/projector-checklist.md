# 프로젝터 환경 체크리스트 — 현장에서 눈에 보이는지 확인

> 이 문서의 목적: 발표장 프로젝터에 **슬라이드와 데모 화면이 제대로 보이는지** 확인하는 구체 체크리스트.
> 사용 방법: 발표장 도착 직후 **순서대로 10~15분** 안에 모든 항목 확인한다. 프로젝터 문제는 발표 시작 전에 반드시 해결되어야 한다.

---

## 🖥️ Step 1 — 하드웨어 연결 확인 (3분)

### 체크 항목

- [ ] **프로젝터 전원 ON**: 리모컨 또는 프로젝터 본체 버튼
- [ ] **프로젝터 준비 완료**: 전원 램프가 녹색으로 안정
- [ ] **HDMI 케이블 연결**: 프로젝터 쪽 HDMI-IN 1번 포트 (보통 기본)
- [ ] **USB-C ↔ HDMI 어댑터 연결**: 맥북이면 어댑터를 USB-C에 꽂고, 반대편을 HDMI 케이블에 연결
- [ ] **노트북 쪽 디스플레이 자동 감지**: 화면 깜빡임 확인

### 문제 해결

**프로젝터에 신호 없음**:
1. 프로젝터 입력 소스 확인: 리모컨 `Input` → `HDMI 1`
2. HDMI 케이블 다른 포트에 꽂아보기
3. 어댑터가 정상인지 확인 (다른 HDMI 기기로 테스트)

**이미지는 나오지만 떨림**:
1. HDMI 케이블 양쪽 완전 체결 확인
2. 어댑터를 천천히 돌려 접촉 개선

---

## ⚙️ Step 2 — 해상도·디스플레이 모드 설정 (3분)

### macOS 기준 (시스템 설정 → 디스플레이)

- [ ] **"모니터 미러링"** (또는 "Mirror Displays") 옵션 **체크**: 노트북 화면과 프로젝터 화면이 동일하게 표시됨
- [ ] **해상도**: 1920 × 1080 (FHD) 또는 1280 × 720 (HD) 추천
- [ ] **주사율**: 60Hz
- [ ] **색상 프로파일**: 기본값 유지 (변경 불필요)

### 체크 방법

청중석에서 봤을 때 다음이 **또렷하게** 보이는지 확인:

- [ ] 슬라이드 제목 (대형 폰트)
- [ ] 본문 텍스트 (중형 폰트)
- [ ] 표 안의 숫자 (**가장 중요** — 실험 결과 수치)
- [ ] 그림의 라벨 (그림 3, 그림 5 등)

### 안 보이면

1. 해상도를 한 단계 낮춤 (1920×1080 → 1280×720)
2. PPT 전체화면 모드(F5)에서 확대 기능 사용
3. 최악의 경우 인쇄본 출력물 활용 (→ `handout/`)

---

## 🎨 Step 3 — 색상·대비 확인 (2분)

프로젝터는 **밝은 실내·어두운 실내에 따라 색이 다르게** 보인다. 현장 조건에서 다음이 보이는지 확인:

### 필수 색상 체크 항목

- [ ] **차트의 4등급 색상 구분** (Enterprise/Premium/Standard/Free): 슬라이드 9 WFQ 차등 그래프
- [ ] **JFI 차트의 색상** (슬라이드 11 근처): 막대 차트에서 1.000과 0.316이 구분되는지
- [ ] **표 안의 강조 색상**: 슬라이드 7에서 WFQ가 빨갛게·굵게 표시된 부분
- [ ] **코드 블록의 가독성** (슬라이드 4의 아키텍처): 배경과 텍스트 대비

### 색상이 안 보이면

1. 프로젝터 리모컨 → Menu → Picture → **명암/밝기** 조정
2. PPT에서 해당 슬라이드 **미리 확인 후 색상 변경** (발표 전까지만)
3. 최악의 경우 "그래프는 인쇄본을 참고해 주시면 감사하겠습니다" 한 줄 멘트

---

## 🔊 Step 4 — 오디오 확인 (1분)

데모에는 오디오가 필요 없지만, 혹시 동영상이 들어갔거나 시스템 알림음이 울리지 않도록 확인.

- [ ] **노트북 시스템 소리 음소거** (메뉴 바 오른쪽 스피커 아이콘)
- [ ] **방해금지 모드 ON** (제어 센터 → 방해금지 모드)
- [ ] **배터리 관련 알림** 최소 비율로 (90% 이상이면 알림 안 울림)
- [ ] **Slack, Mail, Calendar 같은 알림 앱 종료**

---

## 🔄 Step 5 — 슬라이드 전환 테스트 (2분)

- [ ] **PPT 전체화면 진입** (F5 또는 Command + Shift + Return)
- [ ] **슬라이드 1 → 2 → 3 전환 테스트**: 키보드 방향키 또는 스페이스바
- [ ] **슬라이드 애니메이션 동작 확인**: 문제 있으면 애니메이션 끄기
- [ ] **프레젠터 뷰** 확인 (사용 예정이면): 노트북에는 다음 슬라이드·발표 노트, 프로젝터에는 현재 슬라이드

### 프레젠터 뷰 사용 여부

**쓰는 게 좋은 경우**:
- 미리 작성한 발표 노트가 있을 때
- 시간 관리가 필요할 때 (프레젠터 뷰에 타이머 표시)

**안 쓰는 게 좋은 경우**:
- 학부 발표 시간 관리가 단순할 때 (12분 이내)
- 프레젠터 뷰 설정이 꼬이거나 불안정할 때

→ 서민지는 **프레젠터 뷰 없이 풀 스크린**을 추천. 간단하고 실수 확률 낮음.

---

## 💻 Step 6 — 데모 환경 배치 (3분)

데모에서는 PPT ↔ 터미널 ↔ 브라우저 **3종을 전환**해야 한다. 전환이 부드럽도록 미리 배치.

### macOS 기준 설정

- [ ] **터미널 3개 동시 열기**: 작업 디렉토리 `02-implementation`에 위치
  - 터미널 1: 서버 기동용 (아직 실행 안 함)
  - 터미널 2: curl 전송용 (명령어 히스토리에 미리 준비)
  - 터미널 3: Ollama 서버용 (`ollama serve`)
- [ ] **브라우저 시크릿 창**: `http://localhost:3000/dashboard.html` 로드 준비
- [ ] **PPT 전체화면 실행** 중

### 데모 중 전환 방법

- **PPT → 터미널**: `Cmd+Tab` 키로 전환
- **터미널 → 브라우저**: `Cmd+Tab` 반복 키 또는 `Cmd+Tab+방향키`
- **브라우저 → PPT**: 같은 방식

### 실수 방지 팁

- 터미널 내 명령어는 **상방향 화살표**로 이전 명령 불러오기 (직접 타이핑 시 실수 위험)
- `demo-scenario.md`의 시나리오 1, 2 명령을 **순서대로 히스토리에 저장** 해 두기
- Cmd+Tab이 헷갈리면 **트랙패드 3손가락 상방향 스와이프**로 Mission Control 열어 선택

---

## 🔌 Step 7 — 전원·충전 확인 (1분)

- [ ] **노트북 충전기 콘센트 연결**: 발표 내내 충전 상태 유지
- [ ] **배터리 표시 "충전 중"** 확인 (메뉴 바)
- [ ] **충전 케이블이 발에 걸리지 않는 위치**: 동선 점검
- [ ] **배터리 잔량 90% 이상**: 최소 값

### 콘센트가 멀리 있을 때

- 연장 케이블 지참 (미리 준비)
- 안 되면 발표 시작 전 **최대 충전** 후 진행 (15~20분 발표라면 충전 없이도 가능)

---

## 🌡️ Step 8 — 환경 외부 요인 체크 (2분)

### 조명

- [ ] 프로젝터 가까운 **조명이 슬라이드에 직접 비추지 않는지** 확인
- [ ] 너무 밝으면 조명 조절 요청 (담당자에게)

### 소음

- [ ] 에어컨·선풍기 **소리가 심하지 않은지** 확인
- [ ] 옆 방 소음이 들어오면 문 닫기

### 청중 시야

- [ ] **뒤쪽 좌석**에서도 슬라이드가 보이는지 확인 (본인이 직접 뒤로 가서 확인하거나 담당자에게 부탁)
- [ ] 화면 중앙이 청중 정면인지 확인

---

## ✅ 최종 "Go/No-Go" 체크

발표 시작 **5분 전** 최종 점검.

- [ ] 프로젝터에 슬라이드 1(표지)가 정상 표시
- [ ] 슬라이드 12(데모) 슬라이드까지 미리 한 번 넘겨보기 (애니메이션 포함)
- [ ] 서버·터미널·브라우저가 각자 탭에 배치됨
- [ ] 노트북 충전 중
- [ ] 알림·시스템 소리 OFF
- [ ] 물병 가까이에 놓임
- [ ] **시작 자세**: 슬라이드 1 앞에서 대기, 미소, 심호흡

**모든 항목 ✅이면 → 시작**
**1개 이상 ❌이면 → 담당자에게 즉시 요청**

## 🆘 Troubleshooting 요약 (복사·인쇄용)

| 증상 | 원인 후보 | 해결 |
|---|---|---|
| 프로젝터 빈 화면 | 입력 소스 불일치 | 리모컨 Input → HDMI 1 |
| 프로젝터 화면 떨림 | 케이블 접촉 불량 | 양쪽 완전 체결 |
| 색상이 이상함 | 프로젝터 설정 | Menu → Picture → Default |
| 글자가 너무 작음 | 해상도 너무 높음 | 해상도 1280×720으로 낮춤 |
| 화면이 노트북에만 나옴 | 미러링 OFF | 시스템 설정에서 미러링 ON |
| PPT 전체화면 안 됨 | 단축키 충돌 | F5 또는 Cmd+Shift+Return |
| 터미널 폰트 작음 | 기본 크기 | 터미널 Settings → Font → 18pt 이상 |
| 충전 안 됨 | 케이블/콘센트 문제 | 다른 포트, 다른 콘센트 |
