# 01. 프로젝트 개요 (Project Overview)

> **학습 목표:** 이 문서를 읽고 나면 LLM 스케줄러 프로젝트의 전체 그림을 이해할 수 있습니다.

---

## 1. 프로젝트 정의

### 1.1 프로젝트 명칭

**LLM Scheduler: OS 스케줄링 기법을 활용한 LLM API 요청 최적화 시스템**

### 1.2 핵심 문제

현대 AI 서비스에서 ChatGPT, Claude 같은 LLM API를 사용하는 애플리케이션이 급증하면서 다음과 같은 문제가 발생합니다:

1. **비용 폭증:** 무분별한 API 요청으로 비용이 급증
2. **응답 지연:** 대기열 관리 부재로 응답 시간이 불규칙
3. **공정성 부재:** 모든 요청을 동등하게 처리하면 긴급한 요청도 오래 대기
4. **자원 낭비:** 우선순위 없이 처리하면 시스템 자원 활용이 비효율적

### 1.3 해결 방안

**운영체제의 프로세스 스케줄링 알고리즘을 LLM API 요청 관리에 적용**

| OS 개념 | LLM 시스템 적용 |
|---------|---------------|
| 프로세스 (Process) | LLM API 요청 (Request) |
| CPU 시간 | API 호출 권한 |
| 우선순위 (Priority) | 사용자 등급, 요청 긴급도 |
| 스케줄링 알고리즘 | 요청 처리 순서 결정 |

## 4. 핵심 용어 정의

### 4.1 도메인 용어

| 용어 (영어) | 용어 (한국어) | 정의 |
|-----------|-------------|------|
| **LLM Request** | LLM 요청 | 사용자가 보내는 LLM 처리 요청 |
| **Scheduler** | 스케줄러 | 요청 처리 순서를 결정하는 컴포넌트 |
| **Queue** | 큐 | 대기 중인 요청들을 저장하는 자료구조 |
| **Priority** | 우선순위 | 요청의 중요도 (LOW, NORMAL, HIGH, URGENT) |
| **Tenant** | 테넌트 | 멀티테넌트 환경에서의 고객 조직 |
| **Weight** | 가중치 | 테넌트의 자원 할당 비율 |

### 4.2 OS 스케줄링 용어

| 용어 (영어) | 용어 (한국어) | 정의 |
|-----------|-------------|------|
| **FCFS** | 선착순 | First-Come, First-Served |
| **Starvation** | 기아 현상 | 낮은 우선순위 작업이 무한히 대기하는 현상 |
| **Aging** | 에이징 | 대기 시간이 길어질수록 우선순위를 높이는 기법 |
| **MLFQ** | 다단계 피드백 큐 | Multi-Level Feedback Queue |
| **Time Quantum** | 시간 퀀텀 | 각 큐에서 할당된 CPU 시간 |
| **Boosting** | 부스팅 | 모든 작업을 최고 우선순위로 재설정 |
| **WFQ** | 가중치 공정 큐 | Weighted Fair Queuing |
| **GPS** | 일반화된 프로세서 공유 | Generalized Processor Sharing |
| **Virtual Time** | 가상 시간 | WFQ에서 공정성을 계산하기 위한 개념 |

### 4.3 성능 지표

| 용어 (영어) | 용어 (한국어) | 정의 |
|-----------|-------------|------|
| **Wait Time** | 대기 시간 | 요청이 큐에서 대기한 시간 |
| **Processing Time** | 처리 시간 | LLM 응답을 생성하는 데 걸린 시간 |
| **Throughput** | 처리량 | 단위 시간당 완료된 요청 수 |
| **Jain's Fairness Index** | 제인 공정성 지수 | 공정성을 0~1 사이 값으로 표현 |

## 6. 4가지 스케줄링 알고리즘 개요

### 6.1 FCFS (First-Come, First-Served)

**특징:**
- 가장 단순한 알고리즘
- 도착 순서대로 처리
- 베이스라인 비교용

**시간 복잡도:** O(1)

**적용 케이스:** 기본 요청 처리, 성능 비교 기준

### 6.2 Priority Scheduling

**특징:**
- 우선순위 기반 처리
- URGENT > HIGH > NORMAL > LOW
- Aging으로 기아 방지

**시간 복잡도:** O(log n) (우선순위 큐)

**적용 케이스:** 긴급 요청 우선 처리 (고객 문의 등)

### 6.3 MLFQ (Multi-Level Feedback Queue)

**특징:**
- 4단계 큐 (Q0: 1초, Q1: 3초, Q2: 8초, Q3: 무제한)
- 5가지 MLFQ 규칙 구현
- 주기적 Boosting

**시간 복잡도:** O(log n) (여러 우선순위 큐)

**적용 케이스:** 대화형/배치 작업 혼합 환경

### 6.4 WFQ (Weighted Fair Queuing)

**특징:**
- 멀티테넌트 환경 지원
- Virtual Time으로 GPS 근사
- Jain's Fairness Index로 공정성 측정

**시간 복잡도:** O(log n)

**적용 케이스:** SaaS 멀티테넌트 서비스
