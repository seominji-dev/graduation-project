**내용**:
- AI 서비스 급증: ChatGPT, Claude 등 LLM API 기반 애플리케이션 폭발적 증가

  - 비용 폭증: 무분별한 API 요청으로 예산 초과

  - 공정성 부재: 긴급 요청도 대량 작업 뒤에서 대기
  - 자원 낭비: 우선순위 없는 비효율적 처리

**시각 자료**: 문제 상황 다이어그램 (사용자 A, B, C의 요청이 무질서하게 처리되는 모습)

### 슬라이드 7: 4가지 스케줄링 알고리즘

**핵심 메시지**: 본 프로젝트에서 구현한 4가지 알고리즘

**알고리즘 소개**:

1. **FCFS (First-Come, First-Served)**
   - 도착 순서대로 처리
   - 베이스라인 알고리즘

2. **Priority Scheduling**
   - 우선순위 기반 처리
   - Aging으로 기아 방지

3. **MLFQ (Multi-Level Feedback Queue)**
   - 4단계 큐 (Q0~Q3)
   - 응답성과 처리량 동시 달성

4. **WFQ (Weighted Fair Queuing)**
   - 가중치 기반 공정 스케줄링
   - 멀티테넌트 환경 최적화

---

### 슬라이드 8: MLFQ 상세 설명

**핵심 메시지**: MLFQ의 5가지 규칙과 동작 원리

**5가지 규칙**:
- Rule 1: 높은 우선순위 큐 먼저 실행
- Rule 2: 같은 우선순위는 Round-Robin
- Rule 3: 새 작업은 최고 우선순위(Q0)에서 시작
- Rule 4: 타임 슬라이스 초과 시 강등
- Rule 5: 주기적으로 모든 작업을 Q0로 Boost

**큐 구성**:
| 큐 | 타임 퀀텀 | 용도 |
|----|----------|------|
| Q0 | 1초 | 짧은 대화형 요청 |
| Q1 | 3초 | 중간 길이 요청 |
| Q2 | 8초 | 긴 요청 |
| Q3 | 무제한 | 배치 작업 |

**시각 자료**: MLFQ 큐 구조 및 작업 이동 다이어그램

---

### 슬라이드 9: WFQ와 공정성

**핵심 메시지**: WFQ를 통한 멀티테넌트 공정성 보장

**Virtual Time 개념**:
- Virtual Finish Time = Virtual Start Time + (Service Time / Weight)

**테넌트 가중치**:
- Enterprise: 100
- Premium: 50
- Standard: 10
- Free: 1

**Jain's Fairness Index**:
- J = 1.0: 완벽한 공정성
- 본 시스템: 0.95 이상 유지

**시각 자료**: 가중치에 따른 자원 분배 파이 차트

## 구현 상세 (4-5 슬라이드)

### 슬라이드 15: FCFS & Priority 구현

**FCFS 구현**:
- BullMQ 큐의 FIFO 특성 활용
- 모든 작업에 동일 우선순위 부여
- 시간 복잡도: O(1)

**Priority 구현**:
- BullMQ priority 옵션 활용
- 우선순위 변환: (MAX_PRIORITY - priority) * 2
- AgingManager 연동

**코드 하이라이트**:
- Priority 변환 로직
- URGENT(3) -> 0, HIGH(2) -> 2, NORMAL(1) -> 4, LOW(0) -> 6

---

### 슬라이드 16: MLFQ 구현

**핵심 구현 포인트**:

1. **4개 독립 큐**: mlfq-q0, mlfq-q1, mlfq-q2, mlfq-q3
2. **타임 슬라이스 추적**: 각 작업의 CPU 사용량 기록
3. **큐 강등 로직**: 타임 슬라이스 초과 시 다음 레벨로 이동
4. **Boost 구현**: 60초마다 모든 작업을 Q0로 이동

**성능 특성**:
- 대화형 요청: Q0에서 빠르게 처리
- 배치 작업: 점진적 강등 후 Q3에서 안정적 처리
- 공정성: Boost로 장기 대기 방지

---

### 슬라이드 17: WFQ 구현

**Virtual Time 계산**:
- Virtual Finish Time = Virtual Start Time + (Service Time / Weight)

**구현 컴포넌트**:
1. **VirtualTimeTracker**: 전역 가상 시간 관리
2. **TenantRegistry**: 테넌트 가중치 관리
3. **FairnessCalculator**: 공정성 지표 계산

**공정성 보장 원리**:
- 가중치가 높은 테넌트: 낮은 Virtual Finish Time -> 우선 처리
- 가중치가 낮은 테넌트: 높은 Virtual Finish Time -> 지연 처리
- 결과적으로 가중치 비율에 맞는 자원 분배
