# 에이전트 데드락 탐지 시스템

> OS 데드락 이론을 멀티 에이전트 시스템에 적용

## 왜 이 주제?

여러 AI 에이전트가 협업할 때, 서로 상대방의 결과를 기다리는 상황이 생길 수 있다.

예를 들어:
- 에이전트 A: "B가 코드 분석 끝나면 리뷰할게"
- 에이전트 B: "A가 리뷰 기준 정해주면 분석할게"

→ 둘 다 무한 대기... 시스템 멈춤

이거 완전 OS에서 배운 데드락이랑 같은 상황이다.

## 핵심 아이디어

| OS 개념 | 멀티 에이전트에 적용하면 |
|---------|------------------------|
| 프로세스 | AI 에이전트 |
| 자원 | 다른 에이전트의 출력, API 할당량 |
| Wait-For Graph | 에이전트 대기 관계 그래프 |
| 사이클 탐지 | 데드락 발생 확인 |
| Banker's Algorithm | 자원 할당 전 안전성 검사 |

## 구현할 것

1. **에이전트 상태 모니터**: 각 에이전트가 뭘 기다리는지 추적
2. **Wait-For Graph**: 대기 관계를 그래프로 표현
3. **사이클 탐지**: DFS로 데드락 탐지
4. **회복 전략**: 롤백, 강제 종료, 재시작 중 선택

## 데드락 처리 전략

| 전략 | 구현 방식 |
|------|----------|
| 탐지 | Wait-For Graph + 사이클 탐지 |
| 예방 | Banker's Algorithm 변형 |
| 회복 | 피해 최소 에이전트 선택 → 롤백 |

## 기술 스택

```
Express.js, MongoDB, Redis
Socket.io (실시간 모니터링)
Ollama (멀티 에이전트 시뮬레이션)
```

## 예상 일정

- 설계: 1주
- 멀티 에이전트 기본 환경: 2주
- 데드락 탐지 알고리즘: 3주
- 회복 전략 + 실험: 3주

## 왜 이게 좋은지

- OS 데드락 이론의 **클래식한 개념**을 현대적 문제에 적용
- 멀티 에이전트 시스템의 신뢰성/안정성 연구
- 논문 작성 가능: "멀티 AI 에이전트 시스템의 데드락 탐지 및 회복 알고리즘"
