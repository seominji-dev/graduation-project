# 졸업 프로젝트 후보

10개 아이디어 중 OS 교수님 + 1인 개발 가능 + 학술적 가치 기준으로 4개 선별.

## 후보 비교

| 순위 | 프로젝트 | OS 개념 | 난이도 | 추천 이유 |
|:----:|----------|---------|:------:|-----------|
| 1 | [LLM 스케줄러](./candidate-1-llm-scheduler/) | 스케줄링 | 중상 | OS 직접 연관, 정량 실험 가능 |
| 2 | [데드락 탐지](./candidate-3-deadlock-detector/) | 데드락 | 상 | 클래식 OS 개념 적용 |
| 3 | [메모리 관리](./candidate-2-memory-manager/) | 페이징/가상메모리 | 중상 | 에이전트 장기 기억 문제 해결 |
| 4 | [체크포인팅](./candidate-4-checkpointing/) | 체크포인팅 | 중 | 안정적 완성 가능 (백업) |

## 선택 가이드

```
최우선 추천: #1 LLM 스케줄러
├─ OS 스케줄링을 가장 직접적으로 구현
├─ 알고리즘별 비교 실험이 명확함
└─ 논문 스토리가 깔끔함

안전한 선택: #4 체크포인팅
├─ 난이도가 가장 낮음
├─ 예상치 못한 문제 발생 시 전환 가능
└─ 그래도 학술적 의미 있음
```
