# 중간보고서 목차 구조 (Outline)

**총 분량 목표: ~10페이지**

---

## 1. 서론 (~1페이지)

### 1.1 연구 배경
- LLM API 서비스 확산과 다중 사용자 환경 보편화
- FCFS 기반 처리의 한계 (호위 효과, 기아 현상, 공정성 부재)

### 1.2 연구 목적
- OS 스케줄링 알고리즘의 LLM 도메인 적용
- 5가지 알고리즘 구현 및 정량 비교
- JFI 기반 공정성 측정 방법론 제시

### 1.3 연구 범위
- 25-2학기 프로토타입 구현 완료
- 26-1학기 실험 보강 및 분석 심화

## 2. 관련 연구 (~2페이지)

### 2.1 OS 스케줄링 알고리즘

- 핵심 참고: Silberschatz [1], Corbato [2], Arpaci-Dusseau [3], Demers [4]

### 2.3 공정성 측정

- 본 연구의 이중 수준 JFI 적용

## 3. 시스템 설계 (~3페이지)

### 3.1 시스템 개요 및 OS-LLM 대응 관계
- 표: 프로세스↔LLM 요청, CPU 시간↔API 쿼터 등

### 3.2 시스템 아키텍처
- 4계층 구조도 (클라이언트 → API → 스케줄러 엔진 → 저장소)
- 핵심 컴포넌트 설명

### 3.3 스케줄링 알고리즘 설계
- FCFS: 베이스라인
- Priority with Aging: 4단계 우선순위, 기아 방지
- MLFQ: 4단계 피드백 큐, 선점형 시간 슬라이스, 의사코드
- WFQ: GPS 기반, Virtual Finish Time, 이중 JFI

### 3.4 핵심 기술 특징
- 런타임 알고리즘 교체 (Strategy 패턴)
- 기아 방지 메커니즘 (Aging + Boosting)

## 4. 구현 현황 (~2페이지)

### 4.1 기술 스택 및 개발 환경
- Node.js 22, Express.js 4.18, Jest 29.x
- 의존성 2개 (express, uuid), 테스트 307개, 커버리지 99.76%

### 4.2 실험 결과
- 표: 알고리즘별 성능 비교 (100건 기본 실험)
- RQ1: Priority → URGENT 62% 개선
- RQ2: MLFQ → Short 73.78% 개선 (10 시드 다중 실험, p<0.001)
- RQ3: WFQ → Enterprise 5.8x, JFI 0.89-0.98

### 4.3 통계적 검증
- 다중 시드 실험 설계 (10 시드 × 500 요청)
- 95% 신뢰구간, 대응표본 t-검정, Cohen's d
