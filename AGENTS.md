# 홍익대학교 졸업 프로젝트 문서 저장소

## 개요

2025년 홍익대학교 컴퓨터공학과 졸업 프로젝트를 준비하는 저장소입니다.

### 프로젝트 진행 단계

```
1. 리서치 & 아이디어 수집 ✅ 완료
   └── 기존 졸업 프로젝트 분석, AI 트렌드 조사

2. 후보 제안서 작성 (4개) ✅ 완료
   └── 각 주제별 폴더로 분리 관리

3. 프로토타입 구현 ← 현재 단계
   └── 각 후보 프로젝트의 핵심 기능 검증

4. 최종 선정 & 완성
   └── 1개 프로젝트 선별 → 졸업작품 완성
```

## 문서 작성 스타일

### 톤앤매너 (중요!)

- **대학생 3학년 수준**으로 작성 (너무 전문적이거나 너무 쉬우면 안됨)
- AI가 쓴 것 같은 딱딱한 문체 피하기 (자연스러운 한국어 사용)
- 적당히 학술적이되, 읽기 편하게
- "~입니다", "~됩니다" 보다는 상황에 맞게 "~이다", "~된다" 혼용 가능

### 피해야 할 표현

- 과도한 수식어 남발 ("매우 중요한", "핵심적인", "혁신적인" 등 반복 X)
- 영어 용어 과다 사용 (꼭 필요한 기술 용어만)
- 너무 형식적인 문장 구조

### 권장하는 표현

- 구어체와 문어체의 적절한 균형
- 실제로 대학생이 쓸 법한 자연스러운 문장
- 필요할 때만 기술 용어 사용하고, 가능하면 풀어서 설명

## 핵심 기술 도메인

이 프로젝트의 포인트는 **OS 수업에서 배운 개념들을 AI 에이전트에 적용**하는 것:

| OS 개념 | AI에 적용하면 | 예시 |
|---------|--------------|------|
| 스케줄링 | LLM 요청 순서 관리 | #1 LLM 스케줄러 |
| 메모리 관리 | 에이전트 컨텍스트 관리 | #3 메모리 관리 시스템 |
| IPC/동기화 | 에이전트 간 통신 | #2 멀티 에이전트 오케스트레이터 |
| 데드락 | 에이전트 교착 상태 | #7 데드락 탐지 |

### 기술 스택

**필수 (현재 공부 중)**
```
Backend: Node.js + Express
DB: MongoDB
```

**LLM (24GB RAM 맥북에서 로컬 구동 가능해야 함)**
```
로컬 LLM: Ollama + Llama 3.2, Mistral, Phi-3 등
         (8B 파라미터 이하 모델 권장)
API 대안: OpenAI / Claude API (로컬이 어려울 경우)
```

**선택 (필요시 추가)**
```
큐: BullMQ (Redis) - 스케줄링 프로젝트 시
벡터DB: Chroma, LanceDB - RAG 구현 시
```
